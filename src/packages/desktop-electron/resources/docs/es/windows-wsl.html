Windows (WSL)

Ejecuta vanilla-closedcode en Windows con WSL para una mejor experiencia.

Aunque vanilla-closedcode puede ejecutarse directamente en Windows, recomendamos usar Windows Subsystem for Linux (WSL) para obtener la mejor experiencia. WSL ofrece un entorno Linux que funciona perfectamente con las funciones de vanilla-closedcode.

¿Por qué WSL?

WSL ofrece mejor rendimiento del sistema de archivos, soporte completo de terminal y compatibilidad con las herramientas de desarrollo de las que depende vanilla-closedcode.
---

Configuración

  1. Instala WSL

    Si aún no lo hiciste, instala WSL usando la guía oficial de Microsoft.

  2. Instala vanilla-closedcode en WSL

    Cuando WSL esté listo, abre tu terminal de WSL e instala vanilla-closedcode con uno de los métodos de instalación.

    Descargue el instalador desde la página de Releases: github.com/informanellica/vanilla-closedcode/releases

  3. Usa vanilla-closedcode desde WSL

    Ve al directorio de tu proyecto (accede a los archivos de Windows desde /mnt/c/, /mnt/d/, etc.) y ejecuta vanilla-closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

Aplicación de escritorio + servidor WSL

Si prefieres usar la app de escritorio de vanilla-closedcode pero quieres ejecutar el servidor en WSL:

  1. Inicia el servidor en WSL con --hostname 0.0.0.0 para permitir conexiones externas:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Conecta la app de escritorio a http://localhost:4096

Si `localhost` no funciona en tu configuración, conéctate usando la dirección IP de WSL (desde WSL: `hostname -I`) y usa `http://:4096`.
Cuando uses `--hostname 0.0.0.0`, define `CLOSEDCODE_SERVER_PASSWORD` para proteger el servidor. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Cliente web + WSL

Para obtener la mejor experiencia web en Windows:

  1. Ejecuta closedcode web en la terminal de WSL en lugar de PowerShell:

    closedcode web --hostname 0.0.0.0

  2. Accede desde tu navegador de Windows en http://localhost:<port> (vanilla-closedcode muestra la URL)

Ejecutar closedcode web desde WSL garantiza acceso correcto al sistema de archivos e integración con la terminal, manteniéndolo accesible desde tu navegador de Windows.

Acceso a archivos de Windows

WSL puede acceder a todos tus archivos de Windows mediante el directorio /mnt/:

  • Unidad C: → /mnt/c/
  • Unidad D: → /mnt/d/
  • Y así sucesivamente...

Ejemplo:

cd /mnt/c/Users/YourName/Documents/project
closedcode
Para una experiencia más fluida, considera clonar o copiar tu repo al sistema de archivos de WSL (por ejemplo en `~/code/`) y ejecutar vanilla-closedcode ahí.
---

Consejos

  • Ejecuta vanilla-closedcode en WSL para proyectos guardados en unidades de Windows: el acceso a archivos es transparente
  • Usa la extensión WSL de VS Code junto con vanilla-closedcode para un flujo de desarrollo integrado
  • Tu configuración y tus sesiones de vanilla-closedcode se guardan dentro del entorno WSL en ~/.local/share/closedcode/
