CLI

Opciones y comandos de la CLI de vanilla-closedcode.

La CLI de vanilla-closedcode de forma predeterminada inicia el TUI cuando se ejecuta sin ningún argumento.

closedcode

Pero también acepta comandos como se documenta en esta página. Esto le permite interactuar con vanilla-closedcode mediante programación.

closedcode run "Explain how closures work in JavaScript"

tui

Inicie la interfaz de usuario del terminal vanilla-closedcode.

closedcode [project]

Opciones

Opción Corta Descripción
{"--continue"} -c Continuar la última sesión
{"--session"} -s ID de sesión para continuar
{"--fork"} Bifurcar la sesión al continuar (usar con --continue o --session)
{"--prompt"} Aviso de uso
{"--model"} -m Modelo a utilizar en forma de proveedor/modelo
{"--agent"} Agente a utilizar
{"--port"} Puerto para escuchar
{"--hostname"} Nombre de host para escuchar

Comandos

La CLI de vanilla-closedcode también tiene los siguientes comandos.

agent

Administrar agentes para vanilla-closedcode.

closedcode agent [command]

attach

Conecte una terminal a un servidor backend vanilla-closedcode que ya se esté ejecutando y iniciado mediante los comandos serve o web.

closedcode attach [url]

Esto permite usar TUI con un backend remoto vanilla-closedcode. Por ejemplo:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Opciones

Opción Corta Descripción
{"--dir"} Directorio de trabajo para iniciar TUI en
{"--continue"} -c Continuar la última sesión
{"--session"} -s ID de sesión para continuar
{"--fork"} Bifurcar la sesión al continuar (usar con --continue o --session)
{"--password"} -p Contraseña de autenticación básica (predeterminada: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Usuario de autenticación básica (predeterminado: CLOSEDCODE_SERVER_USERNAME u closedcode)

create

Cree un nuevo agente con configuración personalizada.

closedcode agent create

Este comando lo guiará en la creación de un nuevo agente con un mensaje del sistema personalizado y una configuración de herramientas.

list

Enumere todos los agentes disponibles.

closedcode agent list

auth

Comando para administrar credenciales e iniciar sesión para proveedores.

closedcode auth [command]

login

vanilla-closedcode funciona con la lista de proveedores en Models.dev, por lo que puede usar closedcode auth login para configurar las claves API para cualquier proveedor que desee utilizar. Esto se almacena en ~/.local/share/closedcode/auth.json.

closedcode auth login

Cuando se inicia vanilla-closedcode, carga los proveedores desde el archivo de credenciales. Y si hay claves definidas en sus entornos o un archivo .env en su proyecto.

list

Enumera todos los proveedores autenticados tal como están almacenados en el archivo de credenciales.

closedcode auth list

O la versión corta.

closedcode auth ls

logout

Cierra tu sesión de un proveedor eliminándolo del archivo de credenciales.

closedcode auth logout

github

Administre el agente GitHub para la automatización del repositorio.

closedcode github [command]

install

Instale el agente GitHub en su repositorio.

closedcode github install

Esto configura el flujo de trabajo de acciones GitHub necesario y lo guía a través del proceso de configuración. Más información.

run

Ejecute el agente GitHub. Esto se usa normalmente en acciones GitHub.

closedcode github run
Opciones
Opción Descripción
{"--event"} GitHub evento simulado para ejecutar el agente
{"--token"} GitHub token de acceso personal

mcp

Administrar servidores de protocolo de contexto modelo.

closedcode mcp [command]

add

Agregue un servidor MCP a su configuración.

closedcode mcp add

Este comando lo guiará para agregar un servidor MCP local o remoto.

list

Enumere todos los servidores MCP configurados y su estado de conexión.

closedcode mcp list

O utilice la versión corta.

closedcode mcp ls

auth

Autentíquese con un servidor MCP habilitado para OAuth.

closedcode mcp auth [name]

Si no proporciona un nombre de servidor, se le pedirá que seleccione entre los servidores compatibles con OAuth disponibles.

También puede enumerar los servidores compatibles con OAuth y su estado de autenticación.

closedcode mcp auth list

O utilice la versión corta.

closedcode mcp auth ls

logout

Elimine las credenciales OAuth para un servidor MCP.

closedcode mcp logout [name]

debug

Depurar problemas de conexión OAuth para un servidor MCP.

closedcode mcp debug <name>

models

Enumere todos los modelos disponibles de los proveedores configurados.

closedcode models [provider]

Este comando muestra todos los modelos disponibles en sus proveedores configurados en el formato provider/model.

Esto es útil para determinar el nombre exacto del modelo que se usará en su configuración.

Opcionalmente, puede pasar un ID de proveedor para filtrar modelos por ese proveedor.

closedcode models anthropic

Opciones

Opción Descripción
{"--refresh"} Actualizar la caché de modelos desde models.dev
{"--verbose"} Utilice una salida del modelo más detallada (incluye metadatos como costos)

Utilice el indicador --refresh para actualizar la lista de modelos almacenados en caché. Esto es útil cuando se han agregado nuevos modelos a un proveedor y desea verlos en vanilla-closedcode.

closedcode models --refresh

run

Ejecute closedcode en modo no interactivo pasando un mensaje directamente.

closedcode run [message..]

Esto es útil para secuencias de comandos, automatización o cuando desea una respuesta rápida sin iniciar el TUI completo. Por ejemplo.

closedcode run Explain the use of context in Go

También puede conectarse a una instancia closedcode serve en ejecución para evitar tiempos de arranque en frío del servidor MCP en cada ejecución:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Opciones

Opción Corta Descripción
{"--command"} El comando a ejecutar, use mensaje para args
{"--continue"} -c Continuar la última sesión
{"--session"} -s ID de sesión para continuar
{"--fork"} Bifurcar la sesión al continuar (usar con --continue o --session)
{"--share"} Comparte la sesión
{"--model"} -m Modelo a utilizar en forma de proveedor/modelo
{"--agent"} Agente a utilizar
{"--file"} -f Archivo(s) para adjuntar al mensaje
{"--format"} Formato: predeterminado (formateado) o json (eventos JSON sin formato)

{"--attach"} Adjuntar a un servidor closedcode en ejecución (por ejemplo, http://localhost:4096)
{"--password"} -p Contraseña de autenticación básica (predeterminada: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Usuario de autenticación básica (predeterminado: CLOSEDCODE_SERVER_USERNAME u closedcode)
{"--dir"} Directorio de ejecución, o ruta en el servidor remoto al adjuntar

serve

Inicie un servidor vanilla-closedcode sin cabeza para acceso API. Consulte los documentos del servidor para conocer la interfaz HTTP completa.

closedcode serve

Esto inicia un servidor HTTP que proporciona acceso API a la funcionalidad closedcode sin la interfaz TUI. Configure CLOSEDCODE_SERVER_PASSWORD para habilitar la autenticación básica HTTP (el nombre de usuario predeterminado es closedcode).

Opciones

Opción Descripción
{"--port"} Puerto para escuchar
{"--hostname"} Nombre de host para escuchar
{"--mdns"} Habilitar el descubrimiento de mDNS
{"--cors"} Orígenes de navegador adicionales para permitir CORS

session

Administrar vanilla-closedcode sesiones.

closedcode session [command]

list

Enumere todas las sesiones vanilla-closedcode.

closedcode session list
Opciones
Opción Corta Descripción
{"--max-count"} -n Limitar a N sesiones más recientes
{"--format"} Formato de salida: tabla o json (tabla)

stats

Muestre el uso de tokens y las estadísticas de costos para sus sesiones vanilla-closedcode.

closedcode stats

Opciones

Opción Descripción
{"--days"} Mostrar estadísticas de los últimos N días (todo el tiempo)
{"--tools"} Número de herramientas para mostrar (todas)
{"--models"} Mostrar el desglose del uso del modelo (oculto de forma predeterminada). Pase un número para mostrar la parte superior N
{"--project"} Filtrar por proyecto (todos los proyectos, cadena vacía: proyecto actual)

export

Exportar datos de la sesión como JSON.

closedcode export [sessionID]

Si no proporciona una ID de sesión, se le pedirá que seleccione entre las sesiones disponibles.

import

Importe datos de sesión desde un archivo JSON o una URL compartida vanilla-closedcode.

closedcode import <file>

Puede importar desde un archivo local o una URL compartida vanilla-closedcode.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

Inicie un servidor vanilla-closedcode sin cabeza con una interfaz web.

closedcode web

Esto inicia un servidor HTTP y abre un navegador web para acceder a vanilla-closedcode a través de una interfaz web. Configure CLOSEDCODE_SERVER_PASSWORD para habilitar la autenticación básica HTTP (el nombre de usuario predeterminado es closedcode).

Opciones

Opción Descripción
{"--port"} Puerto para escuchar
{"--hostname"} Nombre de host para escuchar
{"--mdns"} Habilitar el descubrimiento de mDNS
{"--cors"} Orígenes de navegador adicionales para permitir CORS

acp

Inicie un servidor ACP (Agent Client Protocol).

closedcode acp

Este comando inicia un servidor ACP que se comunica a través de stdin/stdout usando nd-JSON.

Opciones

Opción Descripción
{"--cwd"} Directorio de trabajo
{"--port"} Puerto para escuchar
{"--hostname"} Nombre de host para escuchar

uninstall

Desinstale vanilla-closedcode y elimine todos los archivos relacionados.

closedcode uninstall

Opciones

Opción Corta Descripción
{"--keep-config"} -c Mantener archivos de configuración
{"--keep-data"} -d Conservar datos de sesión e instantáneas
{"--dry-run"} Mostrar lo que se eliminaría sin eliminar
{"--force"} -f Saltar mensajes de confirmación

upgrade

Actualiza closedcode a la última versión o a una versión específica.

closedcode upgrade [target]

Para actualizar a la última versión.

closedcode upgrade

Para actualizar a una versión específica.

closedcode upgrade v0.1.48

Opciones

Opción Corta Descripción
{"--method"} -m El método de instalación que se utilizó; curl, npm, pnpm, brew

Variable Type Description
CLOSEDCODE_AUTO_SHARE booleano Compartir sesiones automáticamente
CLOSEDCODE_GIT_BASH_PATH cadena Ruta al ejecutable de Git Bash en Windows
CLOSEDCODE_CONFIG cadena Ruta al archivo de configuración
CLOSEDCODE_CONFIG_DIR cadena Ruta al directorio de configuración
CLOSEDCODE_CONFIG_CONTENT cadena Contenido de configuración json en línea
CLOSEDCODE_DISABLE_AUTOUPDATE booleano Deshabilitar las comprobaciones automáticas de actualizaciones
CLOSEDCODE_DISABLE_PRUNE booleano Deshabilitar la poda de datos antiguos
CLOSEDCODE_DISABLE_TERMINAL_TITLE booleano Deshabilitar las actualizaciones automáticas de títulos de terminal
CLOSEDCODE_PERMISSION cadena Configuración de permisos json incorporados
CLOSEDCODE_DISABLE_DEFAULT_PLUGINS booleano Deshabilitar complementos predeterminados
CLOSEDCODE_DISABLE_LSP_DOWNLOAD booleano Deshabilitar las descargas automáticas del servidor LSP
CLOSEDCODE_ENABLE_EXPERIMENTAL_MODELS booleano Habilitar modelos experimentales
CLOSEDCODE_DISABLE_AUTOCOMPACT booleano Deshabilitar la compactación automática de contexto
CLOSEDCODE_DISABLE_CLAUDE_CODE booleano Deshabilitar la lectura desde .claude (mensaje + habilidades)
CLOSEDCODE_DISABLE_CLAUDE_CODE_PROMPT booleano Desactivar lectura ~/.claude/CLAUDE.md
CLOSEDCODE_DISABLE_CLAUDE_CODE_SKILLS booleano Deshabilitar la carga .claude/skills
CLOSEDCODE_DISABLE_MODELS_FETCH booleano Deshabilitar la recuperación de modelos desde fuentes remotas
CLOSEDCODE_FAKE_VCS cadena Proveedor de VCS falso para fines de prueba
CLOSEDCODE_CLIENT cadena Identificador de cliente (por defecto cli)
CLOSEDCODE_ENABLE_EXA booleano Habilitar las herramientas de búsqueda web de Exa
CLOSEDCODE_SERVER_PASSWORD cadena Habilite la autenticación básica para serve/web
CLOSEDCODE_SERVER_USERNAME cadena Anular el nombre de usuario de autenticación básica (predeterminado closedcode)
CLOSEDCODE_MODELS_URL cadena URL personalizada para buscar la configuración de modelos
