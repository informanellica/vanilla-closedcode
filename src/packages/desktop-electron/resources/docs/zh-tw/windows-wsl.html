Windows (WSL)

透過 WSL 在 Windows 上執行 vanilla-closedcode 以獲得最佳體驗。

雖然 vanilla-closedcode 可以直接在 Windows 上執行，但我們推薦使用 Windows Subsystem for Linux (WSL) 以獲得最佳體驗。WSL 提供了一個 Linux 環境，能夠與 vanilla-closedcode 的各項功能無縫配合。

為什麼選擇 WSL？

WSL 提供更出色的檔案系統效能、完整的終端機支援，以及與 vanilla-closedcode 所依賴的開發工具的良好相容性。
---

安裝設定

  1. 安裝 WSL

    如果尚未安裝，請參照 Microsoft 官方指南安裝 WSL。

  2. 在 WSL 中安裝 vanilla-closedcode

    WSL 設定完成後，開啟 WSL 終端機，使用任一安裝方式安裝 vanilla-closedcode。

    curl -fsSL https://github.com/informanellica/vanilla-closedcode/releases | bash

  3. 從 WSL 中使用 vanilla-closedcode

    導航到你的專案目錄（透過 /mnt/c/、/mnt/d/ 等路徑存取 Windows 檔案），然後執行 vanilla-closedcode。

    cd /mnt/c/Users/YourName/project
closedcode

桌面應用程式 + WSL 伺服器

如果你希望使用 vanilla-closedcode 桌面應用程式，同時在 WSL 中執行伺服器：

  1. 在 WSL 中啟動伺服器，新增 --hostname 0.0.0.0 以允許外部連線：

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. 在桌面應用程式中連線到 http://localhost:4096

如果 `localhost` 在你的環境中無法使用，請改用 WSL 的 IP 位址進行連線（在 WSL 中執行：`hostname -I`），使用 `http://:4096`。
使用 `--hostname 0.0.0.0` 時，請設定 `CLOSEDCODE_SERVER_PASSWORD` 以保護伺服器安全。
```bash CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0 ```

Web 客戶端 + WSL

要在 Windows 上獲得最佳的 Web 體驗：

  1. 在 WSL 終端機中執行 closedcode web，而非在 PowerShell 中執行：

    closedcode web --hostname 0.0.0.0

  2. 在 Windows 瀏覽器中存取 http://localhost:<port>（vanilla-closedcode 會輸出該 URL）

從 WSL 中執行 closedcode web 可確保正確的檔案系統存取和終端機整合，同時仍可透過 Windows 瀏覽器進行存取。

存取 Windows 檔案

WSL 可以透過 /mnt/ 目錄存取你的所有 Windows 檔案：

  • C: 磁碟 → /mnt/c/
  • D: 磁碟 → /mnt/d/
  • 其他磁碟以此類推...

範例：

cd /mnt/c/Users/YourName/Documents/project
closedcode
為了獲得更流暢的體驗，建議將儲存庫克隆或複製到 WSL 檔案系統中（例如 `~/code/` 目錄下），然後在該位置執行 vanilla-closedcode。
---

使用技巧

  • 對於儲存在 Windows 磁碟上的專案，在 WSL 中執行 vanilla-closedcode 即可無縫存取檔案
  • 搭配 VS Code 的 WSL 擴充套件 使用 vanilla-closedcode，打造一體化的開發工作流程
  • vanilla-closedcode 的設定和工作階段資料儲存在 WSL 環境中的 ~/.local/share/closedcode/
