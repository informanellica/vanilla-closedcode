CLI

vanilla-closedcode CLI 選項和指令。

vanilla-closedcode CLI 在不帶任何參數執行時，預設啟動 TUI。

closedcode

但它也接受本頁面中記錄的指令，使您可以透過程式方式與 vanilla-closedcode 進行互動。

closedcode run "Explain how closures work in JavaScript"

tui

啟動 vanilla-closedcode 終端機使用者介面。

closedcode [project]

旗標

旗標 簡寫 說明
{"--continue"} -c 繼續上一個工作階段
{"--session"} -s 要繼續的工作階段 ID
{"--fork"} 繼續時分岔工作階段（與 --continue 或 --session 搭配使用）
{"--prompt"} 要使用的提示詞
{"--model"} -m 要使用的模型，格式為 provider/model
{"--agent"} 要使用的代理
{"--port"} 監聽連接埠
{"--hostname"} 監聽主機名稱

指令

vanilla-closedcode CLI 還提供以下指令。

agent

管理 vanilla-closedcode 的代理。

closedcode agent [command]

attach

將終端機連接到已透過 serve 或 web 指令啟動的 vanilla-closedcode 後端伺服器。

closedcode attach [url]

這允許將 TUI 與遠端 vanilla-closedcode 後端搭配使用。例如：

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

旗標

旗標 簡寫 說明
{"--dir"} 啟動 TUI 的工作目錄
{"--continue"} -c 繼續上一個工作階段
{"--session"} -s 要繼續的工作階段 ID
{"--fork"} 繼續時分支工作階段（與 --continue 或 --session 一起使用）
{"--password"} -p 基本驗證密碼（預設使用 CLOSEDCODE_SERVER_PASSWORD）
{"--username"} -u 基本驗證使用者名稱（預設使用 CLOSEDCODE_SERVER_USERNAME 或 closedcode）

create

使用自訂設定建立新的代理。

closedcode agent create

此指令將引導您使用自訂系統提示詞和工具設定來建立新的代理。

list

列出所有可用的代理。

closedcode agent list

auth

管理供應商的憑證和登入資訊的指令。

closedcode auth [command]

login

vanilla-closedcode 基於 Models.dev 的供應商列表運作，因此您可以使用 closedcode auth login 為任何想要使用的供應商設定 API 金鑰。金鑰儲存在 ~/.local/share/closedcode/auth.json 中。

closedcode auth login

vanilla-closedcode 啟動時會從憑證檔案載入供應商資訊，同時也會載入環境變數或專案中 .env 檔案中定義的金鑰。

list

列出憑證檔案中儲存的所有已認證供應商。

closedcode auth list

或使用簡寫版本。

closedcode auth ls

logout

從憑證檔案中清除供應商資訊以完成登出。

closedcode auth logout

github

管理用於儲存庫自動化的 GitHub 代理。

closedcode github [command]

install

在您的儲存庫中安裝 GitHub 代理。

closedcode github install

此指令會設定必要的 GitHub Actions 工作流程並引導您完成設定過程。了解更多。

run

執行 GitHub 代理。通常在 GitHub Actions 中使用。

closedcode github run
旗標
旗標 說明
{"--event"} 用於執行代理的 GitHub 模擬事件
{"--token"} GitHub 個人存取權杖

mcp

管理 Model Context Protocol 伺服器。

closedcode mcp [command]

add

將 MCP 伺服器新增到您的設定中。

closedcode mcp add

此指令將引導您新增本地或遠端 MCP 伺服器。

list

列出所有已設定的 MCP 伺服器及其連線狀態。

closedcode mcp list

或使用簡寫版本。

closedcode mcp ls

auth

對支援 OAuth 的 MCP 伺服器進行認證。

closedcode mcp auth [name]

如果您不提供伺服器名稱，系統將提示您從可用的支援 OAuth 的伺服器中進行選擇。

您還可以列出支援 OAuth 的伺服器及其認證狀態。

closedcode mcp auth list

或使用簡寫版本。

closedcode mcp auth ls

logout

移除 MCP 伺服器的 OAuth 憑證。

closedcode mcp logout [name]

debug

偵錯 MCP 伺服器的 OAuth 連線問題。

closedcode mcp debug <name>

models

列出已設定供應商的所有可用模型。

closedcode models [provider]

此指令以 provider/model 的格式顯示所有已設定供應商中可用的模型。

這對於確定在設定檔中使用的確切模型名稱非常有用。

您可以選擇傳入供應商 ID 來按供應商篩選模型。

closedcode models anthropic

旗標

旗標 說明
{"--refresh"} 從 models.dev 重新整理模型快取
{"--verbose"} 使用更詳細的模型輸出（包含費用等中繼資料）

使用 --refresh 旗標可以更新快取的模型列表。當供應商新增了模型並且您希望在 vanilla-closedcode 中看到它們時，此功能非常有用。

closedcode models --refresh

run

以非互動模式執行 vanilla-closedcode，直接傳入提示詞。

closedcode run [message..]

這對於指令碼編寫、自動化或無需啟動完整 TUI 即可快速取得答案的情境非常有用。例如：

closedcode run Explain the use of context in Go

您還可以連接到正在執行的 closedcode serve 實例，以避免每次執行時 MCP 伺服器的冷啟動時間：

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

旗標

旗標 簡寫 說明
{"--command"} 要執行的指令，使用 message 作為參數
{"--continue"} -c 繼續上一個工作階段
{"--session"} -s 要繼續的工作階段 ID
{"--fork"} 繼續時分岔工作階段（與 --continue 或 --session 搭配使用）
{"--share"} 分享工作階段
{"--model"} -m 要使用的模型，格式為 provider/model
{"--agent"} 要使用的代理
{"--file"} -f 附加到訊息的檔案
{"--format"} 格式：default（格式化輸出）或 json（原始 JSON 事件）
{"--title"} 工作階段標題（未提供值時使用截斷的提示詞）
{"--attach"} 連接到正在執行的 closedcode 伺服器（例如 http://localhost:4096）
{"--password"} -p 基本驗證密碼（預設使用 CLOSEDCODE_SERVER_PASSWORD）
{"--username"} -u 基本驗證使用者名稱（預設使用 CLOSEDCODE_SERVER_USERNAME 或 closedcode）
{"--dir"} 執行目錄，或附加時遠端伺服器上的路徑
{"--variant"} 模型變體（特定於提供者的推理級別）
{"--thinking"} 顯示思考區塊
{"--port"} 本地伺服器連接埠（預設為隨機連接埠）

serve

啟動無介面的 vanilla-closedcode 伺服器以提供 API 存取。查看伺服器文件了解完整的 HTTP 介面。

closedcode serve

此指令啟動一個 HTTP 伺服器，提供對 vanilla-closedcode 功能的 API 存取，無需 TUI 介面。設定 CLOSEDCODE_SERVER_PASSWORD 可啟用 HTTP 基本認證（使用者名稱預設為 closedcode）。

旗標

旗標 說明
{"--port"} 監聽連接埠
{"--hostname"} 監聽主機名稱
{"--mdns"} 啟用 mDNS 探索
{"--cors"} 允許 CORS 的額外瀏覽器來源

session

管理 vanilla-closedcode 工作階段。

closedcode session [command]

list

列出所有 vanilla-closedcode 工作階段。

closedcode session list
旗標
旗標 簡寫 說明
{"--max-count"} -n 限制為最近 N 個工作階段
{"--format"} 輸出格式：table 或 json（預設 table）

stats

顯示 vanilla-closedcode 工作階段的 Token 用量和費用統計資訊。

closedcode stats

旗標

旗標 說明
{"--days"} 顯示最近 N 天的統計資訊（預設為所有時間）
{"--tools"} 顯示的工具數量（預設為全部）
{"--models"} 顯示模型用量明細（預設隱藏）。傳入數字可顯示前 N 個
{"--project"} 按專案篩選（預設為所有專案，傳入空字串表示當前專案）

export

將工作階段資料匯出為 JSON。

closedcode export [sessionID]

如果您不提供工作階段 ID，系統將提示您從可用的工作階段中進行選擇。

import

從 JSON 檔案或 vanilla-closedcode 分享連結匯入工作階段資料。

closedcode import <file>

您可以從本地檔案或 vanilla-closedcode 分享連結匯入。

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

啟動帶有 Web 介面的無介面 vanilla-closedcode 伺服器。

closedcode web

此指令啟動一個 HTTP 伺服器並開啟瀏覽器，透過 Web 介面存取 vanilla-closedcode。設定 CLOSEDCODE_SERVER_PASSWORD 可啟用 HTTP 基本認證（使用者名稱預設為 closedcode）。

旗標

旗標 說明
{"--port"} 監聽連接埠
{"--hostname"} 監聽主機名稱
{"--mdns"} 啟用 mDNS 探索
{"--cors"} 允許 CORS 的額外瀏覽器來源

acp

啟動 ACP（Agent Client Protocol）伺服器。

closedcode acp

此指令啟動一個透過 stdin/stdout 使用 nd-JSON 進行通訊的 ACP 伺服器。

旗標

旗標 說明
{"--cwd"} 工作目錄
{"--port"} 監聽連接埠
{"--hostname"} 監聽主機名稱

uninstall

解除安裝 vanilla-closedcode 並刪除所有相關檔案。

closedcode uninstall

旗標

旗標 簡寫 說明
{"--keep-config"} -c 保留設定檔
{"--keep-data"} -d 保留工作階段資料和快照
{"--dry-run"} 顯示將被刪除的內容但不實際刪除
{"--force"} -f 跳過確認提示

upgrade

環境變數

vanilla-closedcode 可以透過環境變數進行設定。

變數 類型 說明
CLOSEDCODE_AUTO_SHARE boolean 自動分享工作階段
CLOSEDCODE_GIT_BASH_PATH string Windows 上 Git Bash 可執行檔的路徑
CLOSEDCODE_CONFIG string 設定檔路徑
CLOSEDCODE_TUI_CONFIG string TUI 設定檔路徑
CLOSEDCODE_CONFIG_DIR string 設定目錄路徑
CLOSEDCODE_CONFIG_CONTENT string 內嵌 JSON 設定內容
CLOSEDCODE_DISABLE_AUTOUPDATE boolean 停用自動更新檢查
CLOSEDCODE_DISABLE_PRUNE boolean 停用舊資料清理
CLOSEDCODE_DISABLE_TERMINAL_TITLE boolean 停用自動終端機標題更新
CLOSEDCODE_PERMISSION string 內嵌 JSON 權限設定
CLOSEDCODE_DISABLE_DEFAULT_PLUGINS boolean 停用預設外掛程式
CLOSEDCODE_DISABLE_LSP_DOWNLOAD boolean 停用 LSP 伺服器自動下載
CLOSEDCODE_ENABLE_EXPERIMENTAL_MODELS boolean 啟用實驗性模型
CLOSEDCODE_DISABLE_AUTOCOMPACT boolean 停用自動上下文壓縮
CLOSEDCODE_DISABLE_CLAUDE_CODE boolean 停用讀取 .claude（提示詞 + 技能）
CLOSEDCODE_DISABLE_CLAUDE_CODE_PROMPT boolean 停用讀取 ~/.claude/CLAUDE.md
CLOSEDCODE_DISABLE_CLAUDE_CODE_SKILLS boolean 停用載入 .claude/skills
CLOSEDCODE_DISABLE_MODELS_FETCH boolean 停用從遠端來源擷取模型
CLOSEDCODE_FAKE_VCS string 用於測試目的的模擬 VCS 供應商
CLOSEDCODE_CLIENT string 用戶端識別碼（預設為 cli）
CLOSEDCODE_ENABLE_EXA boolean 啟用 Exa 網路搜尋工具
CLOSEDCODE_SERVER_PASSWORD string 為 serve/web 啟用基本認證
CLOSEDCODE_SERVER_USERNAME string 覆寫基本認證使用者名稱（預設為 closedcode）
CLOSEDCODE_MODELS_URL string 自訂模型設定擷取 URL
