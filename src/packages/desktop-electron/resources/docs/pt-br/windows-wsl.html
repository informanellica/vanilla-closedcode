Windows (WSL)

Execute o closedcode no Windows com WSL para melhor experiência.

Embora o closedcode possa rodar direto no Windows, recomendamos usar Windows Subsystem for Linux (WSL) para a melhor experiência. O WSL oferece um ambiente Linux que funciona de forma integrada com os recursos do closedcode.

Por que WSL?

O WSL oferece melhor desempenho de sistema de arquivos, suporte completo a terminal e compatibilidade com as ferramentas de desenvolvimento das quais o closedcode depende.
---

Configuração

  1. Instale o WSL

    Se ainda não instalou, instale o WSL usando o guia oficial da Microsoft.

  2. Instale o closedcode no WSL

    Depois de configurar o WSL, abra o terminal do WSL e instale o closedcode usando um dos métodos de instalação.

    curl -fsSL https://github.com/informanellica/vanilla-closedcode/releases | bash

  3. Use o closedcode pelo WSL

    Vá para o diretório do seu projeto (acesse arquivos do Windows via /mnt/c/, /mnt/d/ etc.) e execute o closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

App desktop + servidor WSL

Se você prefere usar o app desktop do closedcode, mas quer rodar o servidor no WSL:

  1. Inicie o servidor no WSL com --hostname 0.0.0.0 para permitir conexões externas:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Conecte o app desktop em http://localhost:4096

Se `localhost` não funcionar no seu ambiente, conecte usando o IP do WSL (no WSL: `hostname -I`) e use `http://:4096`.
Ao usar `--hostname 0.0.0.0`, defina `CLOSEDCODE_SERVER_PASSWORD` para proteger o servidor. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Cliente web + WSL

Para a melhor experiência web no Windows:

  1. Execute closedcode web no terminal WSL em vez do PowerShell:

    closedcode web --hostname 0.0.0.0

  2. Acesse pelo navegador do Windows em http://localhost:<port> (o closedcode mostra a URL)

Executar closedcode web a partir do WSL garante acesso correto ao sistema de arquivos e integração com o terminal, continuando acessível no navegador do Windows.

Acessar arquivos do Windows

O WSL pode acessar todos os arquivos do Windows pelo diretório /mnt/:

  • Unidade C: → /mnt/c/
  • Unidade D: → /mnt/d/
  • E assim por diante...

Exemplo:

cd /mnt/c/Users/YourName/Documents/project
closedcode
Para uma experiência mais fluida, considere clonar/copiar seu repositório para o sistema de arquivos do WSL (por exemplo em `~/code/`) e executar o closedcode por lá.
---

Dicas

  • Mantenha o closedcode rodando no WSL para projetos armazenados em unidades do Windows - o acesso aos arquivos fica fluido
  • Use a extensão WSL do VS Code junto com o closedcode para um fluxo de desenvolvimento integrado
  • Sua configuração e suas sessões do closedcode ficam armazenadas no ambiente WSL em ~/.local/share/closedcode/
