CLI

Opzioni e comandi della CLI di vanilla-closedcode.

La CLI di vanilla-closedcode, per impostazione predefinita, avvia la TUI quando viene eseguita senza argomenti.

closedcode

Ma accetta anche i comandi documentati in questa pagina. Questo ti permette di interagire con vanilla-closedcode in modo programmatico.

closedcode run "Explain how closures work in JavaScript"

tui

Avvia l'interfaccia testuale di vanilla-closedcode (interfaccia utente da terminale).

closedcode [project]

Flag

Flag Breve Descrizione
{"--continue"} -c Continua l'ultima sessione
{"--session"} -s ID sessione da continuare
{"--fork"} Duplica la sessione quando continui (usa con --continue o --session)
{"--prompt"} Prompt da usare
{"--model"} -m Modello nel formato provider/model
{"--agent"} Agente da usare
{"--port"} Porta su cui mettersi in ascolto
{"--hostname"} Hostname su cui mettersi in ascolto

Comandi

La CLI di vanilla-closedcode include anche i seguenti comandi.

agent

Gestisci gli agenti per vanilla-closedcode.

closedcode agent [command]

attach

Collega un terminale a un backend server di vanilla-closedcode già in esecuzione avviato tramite i comandi serve o web.

closedcode attach [url]

Questo consente di usare la TUI con un backend vanilla-closedcode remoto. Per esempio:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In un altro terminale, collega la TUI al backend in esecuzione
closedcode attach http://10.20.30.40:4096

Flag

Flag Breve Descrizione
{"--dir"} Working directory in cui avviare la TUI
{"--continue"} -c Continua l'ultima sessione
{"--session"} -s ID sessione da continuare
{"--fork"} Crea un fork della sessione durante la continuazione (usa con --continue o --session)
{"--password"} -p Password per l'autenticazione di base (predefinita: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Nome utente per l'autenticazione di base (predefinito: CLOSEDCODE_SERVER_USERNAME o closedcode)

create

Crea un nuovo agente con configurazione personalizzata.

closedcode agent create

Questo comando ti guida nella creazione di un nuovo agente con un system prompt personalizzato e configurazione degli strumenti.

list

Elenca tutti gli agenti disponibili.

closedcode agent list

auth

Comando per gestire le credenziali e il login dei provider.

closedcode auth [command]

login

vanilla-closedcode si basa sull'elenco provider di Models.dev, quindi puoi usare closedcode auth login per configurare le API key per qualunque provider tu voglia usare. Le credenziali vengono salvate in ~/.local/share/closedcode/auth.json.

closedcode auth login

Quando vanilla-closedcode si avvia, carica i provider dal file delle credenziali e, se presenti, anche eventuali key definite nell'ambiente o in un file .env nel progetto.

list

Elenca tutti i provider autenticati come salvati nel file delle credenziali.

closedcode auth list

Oppure la versione corta.

closedcode auth ls

logout

Esegue il logout da un provider rimuovendolo dal file delle credenziali.

closedcode auth logout

github

Gestisci l'agente GitHub per l'automazione dei repository.

closedcode github [command]

install

Installa l'agente GitHub nel tuo repository.

closedcode github install

Questo configura il workflow GitHub Actions necessario e ti guida nel processo di configurazione. Scopri di più.

run

Esegui l'agente GitHub. Tipicamente usato in GitHub Actions.

closedcode github run
Flag
Flag Descrizione
{"--event"} Evento GitHub mock per cui eseguire l'agente
{"--token"} GitHub personal access token

mcp

Gestisci i server Model Context Protocol.

closedcode mcp [command]

add

Aggiungi un server MCP alla tua configurazione.

closedcode mcp add

Questo comando ti guida nell'aggiunta di un server MCP locale o remoto.

list

Elenca tutti i server MCP configurati e il loro stato di connessione.

closedcode mcp list

Oppure la versione corta.

closedcode mcp ls

auth

Autentica con un server MCP con OAuth abilitato.

closedcode mcp auth [name]

Se non fornisci un nome server, ti verrà chiesto di selezionare tra i server OAuth-capable disponibili.

Puoi anche elencare i server OAuth-capable e il loro stato di autenticazione.

closedcode mcp auth list

Oppure la versione corta.

closedcode mcp auth ls

logout

Rimuovi le credenziali OAuth per un server MCP.

closedcode mcp logout [name]

debug

Esegui debug di problemi di connessione OAuth per un server MCP.

closedcode mcp debug <name>

models

Elenca tutti i modelli disponibili dai provider configurati.

closedcode models [provider]

Questo comando mostra tutti i modelli disponibili tra i provider configurati nel formato provider/model.

È utile per capire l'esatto nome del modello da usare nella config.

Puoi anche passare opzionalmente un ID provider per filtrare i modelli a quel provider.

closedcode models anthropic

Flag

Flag Descrizione
{"--refresh"} Aggiorna la cache modelli da models.dev
{"--verbose"} Output più verboso (include metadati come i costi)

Usa --refresh per aggiornare l'elenco modelli in cache. È utile quando nuovi modelli vengono aggiunti a un provider e vuoi vederli in vanilla-closedcode.

closedcode models --refresh

run

Esegui closedcode in modalità non interattiva passando un prompt direttamente.

closedcode run [message..]

È utile per scripting, automazione o quando vuoi una risposta rapida senza avviare la TUI completa. Per esempio:

closedcode run Explain the use of context in Go

Puoi anche collegarti a una istanza closedcode serve già in esecuzione per evitare i cold boot dei server MCP ad ogni esecuzione:

# Avvia un server headless in un terminale
closedcode serve

# In un altro terminale, esegui comandi che si collegano ad esso
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Flag

Flag Breve Descrizione
{"--command"} Il comando da eseguire; usa message per gli argomenti
{"--continue"} -c Continua l'ultima sessione
{"--session"} -s ID sessione da continuare
{"--fork"} Duplica la sessione quando continui (usa con --continue o --session)
{"--share"} Condividi la sessione
{"--model"} -m Modello nel formato provider/model
{"--agent"} Agente da usare
{"--file"} -f File da allegare al messaggio
{"--format"} Formato: default (formattato) o json (eventi JSON grezzi)
{"--title"} Titolo sessione (usa prompt troncato se non viene fornito un valore)
{"--attach"} Attach a un server closedcode in esecuzione (es. http://localhost:4096)
{"--password"} -p Password per l'autenticazione di base (predefinita: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Nome utente per l'autenticazione di base (predefinito: CLOSEDCODE_SERVER_USERNAME o closedcode)
{"--dir"} Directory di esecuzione, o percorso sul server remoto durante il collegamento
{"--variant"} Variante del modello (sforzo di ragionamento specifico del provider)
{"--thinking"} Mostra blocchi di pensiero
{"--port"} Porta per il server locale (di default una porta casuale)

serve

Avvia un server vanilla-closedcode headless per accesso via API. Vedi le server docs per l'interfaccia HTTP completa.

closedcode serve

Avvia un server HTTP che espone accesso API alle funzionalità di closedcode senza la TUI. Imposta CLOSEDCODE_SERVER_PASSWORD per abilitare HTTP basic auth (username di default closedcode).

Flag

Flag Descrizione
{"--port"} Porta su cui mettersi in ascolto
{"--hostname"} Hostname su cui mettersi in ascolto
{"--mdns"} Abilita discovery mDNS
{"--cors"} Origin browser addizionali per consentire CORS

session

Gestisci le sessioni vanilla-closedcode.

closedcode session [command]

list

Elenca tutte le sessioni vanilla-closedcode.

closedcode session list
Flag
Flag Breve Descrizione
{"--max-count"} -n Limita alle N sessioni più recenti
{"--format"} Formato output: table o json (table)

stats

Mostra statistiche di utilizzo token e costo per le sessioni vanilla-closedcode.

closedcode stats

Flag

Flag Descrizione
{"--days"} Mostra statistiche per gli ultimi N giorni (all time)
{"--tools"} Numero di strumenti da mostrare (all)
{"--models"} Mostra breakdown di utilizzo modelli (nascosto di default). Passa un numero per top N
{"--project"} Filtra per progetto (tutti i progetti; stringa vuota: progetto corrente)

export

Esporta i dati di sessione come JSON.

closedcode export [sessionID]

Se non fornisci un ID sessione, ti verrà chiesto di selezionare tra le sessioni disponibili.

import

Importa i dati di sessione da un file JSON o da un URL di condivisione vanilla-closedcode.

closedcode import <file>

Puoi importare da un file locale o da un URL di condivisione vanilla-closedcode.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

Avvia un server vanilla-closedcode headless con interfaccia web.

closedcode web

Avvia un server HTTP e apre un browser per accedere a vanilla-closedcode tramite interfaccia web. Imposta CLOSEDCODE_SERVER_PASSWORD per abilitare HTTP basic auth (username di default closedcode).

Flag

Flag Descrizione
{"--port"} Porta su cui mettersi in ascolto
{"--hostname"} Hostname su cui mettersi in ascolto
{"--mdns"} Abilita discovery mDNS
{"--cors"} Origin browser addizionali per consentire CORS

acp

Avvia un server ACP (Agent Client Protocol).

closedcode acp

Questo comando avvia un server ACP che comunica via stdin/stdout usando nd-JSON.

Flag

Flag Descrizione
{"--cwd"} Directory di lavoro
{"--port"} Porta su cui mettersi in ascolto
{"--hostname"} Hostname su cui mettersi in ascolto

uninstall

Disinstalla vanilla-closedcode e rimuove tutti i file correlati.

closedcode uninstall

Flag

Flag Breve Descrizione
{"--keep-config"} -c Mantieni i file di configurazione
{"--keep-data"} -d Mantieni dati di sessione e snapshot
{"--dry-run"} Mostra cosa verrebbe rimosso senza rimuovere
{"--force"} -f Salta le richieste di conferma

upgrade

Aggiorna closedcode all'ultima versione o a una versione specifica.

closedcode upgrade [target]

Per aggiornare all'ultima versione:

closedcode upgrade

Per aggiornare a una versione specifica:

closedcode upgrade v0.1.48

Flag

Flag Breve Descrizione
{"--method"} -m Metodo di installazione usato: curl, npm, pnpm, brew

Variabili d'ambiente

vanilla-closedcode può essere configurato tramite variabili d'ambiente.

Variabile Tipo Descrizione
CLOSEDCODE_AUTO_SHARE boolean Condivide automaticamente le sessioni
CLOSEDCODE_GIT_BASH_PATH string Percorso all'eseguibile Git Bash su Windows
CLOSEDCODE_CONFIG string Percorso al file di configurazione
CLOSEDCODE_TUI_CONFIG string Percorso al file di configurazione TUI
CLOSEDCODE_CONFIG_DIR string Percorso alla directory di configurazione
CLOSEDCODE_CONFIG_CONTENT string Contenuto JSON di config inline
CLOSEDCODE_DISABLE_AUTOUPDATE boolean Disabilita i controlli automatici di aggiornamento
CLOSEDCODE_DISABLE_PRUNE boolean Disabilita la potatura dei dati vecchi
CLOSEDCODE_DISABLE_TERMINAL_TITLE boolean Disabilita aggiornamenti automatici del titolo terminale
CLOSEDCODE_PERMISSION string Config permessi JSON inline
CLOSEDCODE_DISABLE_DEFAULT_PLUGINS boolean Disabilita i plugin di default
CLOSEDCODE_DISABLE_LSP_DOWNLOAD boolean Disabilita download automatico dei server LSP
CLOSEDCODE_ENABLE_EXPERIMENTAL_MODELS boolean Abilita modelli sperimentali
CLOSEDCODE_DISABLE_AUTOCOMPACT boolean Disabilita compaction automatica del contesto
CLOSEDCODE_DISABLE_CLAUDE_CODE boolean Disabilita lettura da .claude (prompt + skill)
CLOSEDCODE_DISABLE_CLAUDE_CODE_PROMPT boolean Disabilita lettura di ~/.claude/CLAUDE.md
CLOSEDCODE_DISABLE_CLAUDE_CODE_SKILLS boolean Disabilita caricamento di .claude/skills
CLOSEDCODE_DISABLE_MODELS_FETCH boolean Disabilita fetch dei modelli da fonti remote
CLOSEDCODE_FAKE_VCS string Provider VCS finto per scopi di test
CLOSEDCODE_CLIENT string Identificatore client (default cli)
CLOSEDCODE_ENABLE_EXA boolean Abilita gli strumenti di web search Exa
CLOSEDCODE_SERVER_PASSWORD string Abilita basic auth per serve/web
CLOSEDCODE_SERVER_USERNAME string Sovrascrive lo username basic auth (default closedcode)
CLOSEDCODE_MODELS_URL string URL personalizzato per recuperare la configurazione modelli
