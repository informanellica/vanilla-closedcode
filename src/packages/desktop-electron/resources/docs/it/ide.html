IDE

L'estensione vanilla-closedcode per VS Code, Cursor e altri IDE

vanilla-closedcode si integra con VS Code, Cursor o qualunque IDE che supporti un terminale. Basta eseguire closedcode nel terminale per iniziare.

Utilizzo

  • Avvio rapido: usa Cmd+Esc (Mac) o Ctrl+Esc (Windows/Linux) per aprire vanilla-closedcode in una vista terminale divisa, oppure per mettere a fuoco una sessione esistente se e gia in esecuzione.
  • Nuova sessione: usa Cmd+Shift+Esc (Mac) o Ctrl+Shift+Esc (Windows/Linux) per avviare una nuova sessione di vanilla-closedcode, anche se ne esiste gia una aperta. Puoi anche cliccare il pulsante vanilla-closedcode nell'interfaccia.
  • Consapevolezza del contesto: condividi automaticamente con vanilla-closedcode la selezione corrente o la scheda attiva.
  • Scorciatoie per riferimenti file: usa Cmd+Option+K (Mac) o Alt+Ctrl+K (Linux/Windows) per inserire riferimenti a file. Per esempio, @File#L37-42.

Installazione

Per installare vanilla-closedcode su VS Code e fork popolari come Cursor, Windsurf, VSCodium:

  1. Apri VS Code
  2. Apri il terminale integrato
  3. Esegui closedcode - l'estensione si installa automaticamente

Se invece vuoi usare il tuo IDE quando esegui /editor o /export dalla TUI, devi impostare export EDITOR="code --wait". Scopri di piu.

Installazione manuale

Cerca vanilla-closedcode nel Marketplace delle estensioni e clicca Install.

Risoluzione dei problemi

Se l'estensione non si installa automaticamente:

  • Assicurati di eseguire closedcode nel terminale integrato.
  • Verifica che la CLI del tuo IDE sia installata:
    • Per VS Code: comando code
    • Per Cursor: comando cursor
    • Per Windsurf: comando windsurf
    • Per VSCodium: comando codium
    • In caso contrario, esegui Cmd+Shift+P (Mac) o Ctrl+Shift+P (Windows/Linux) e cerca "Shell Command: Install 'code' command in PATH" (o l'equivalente per il tuo IDE)
  • Assicurati che VS Code abbia i permessi per installare estensioni
