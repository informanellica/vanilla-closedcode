Windows (WSL)

Esegui vanilla-closedcode su Windows con WSL per la migliore esperienza.

Anche se vanilla-closedcode puo essere eseguito direttamente su Windows, consigliamo di usare Windows Subsystem for Linux (WSL) per la migliore esperienza. WSL fornisce un ambiente Linux che funziona in modo ottimale con le funzionalita di vanilla-closedcode.

Perche WSL?

WSL offre prestazioni migliori del file system, supporto completo del terminale e compatibilita con gli strumenti di sviluppo su cui vanilla-closedcode fa affidamento.
---

Configurazione

  1. Installa WSL

    Se non l'hai ancora fatto, installa WSL seguendo la guida ufficiale Microsoft.

  2. Installa vanilla-closedcode in WSL

    Dopo aver configurato WSL, apri il terminale WSL e installa vanilla-closedcode scaricando l'ultima versione dalla pagina Releases: github.com/informanellica/vanilla-closedcode/releases.

  3. Usa vanilla-closedcode da WSL

    Vai nella directory del progetto (accedi ai file Windows tramite /mnt/c/, /mnt/d/, ecc.) ed esegui vanilla-closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

App desktop + server WSL

Se preferisci usare l'app desktop di vanilla-closedcode ma vuoi eseguire il server in WSL:

  1. Avvia il server in WSL con --hostname 0.0.0.0 per consentire connessioni esterne:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Collega l'app desktop a http://localhost:4096

Se `localhost` non funziona nella tua configurazione, usa l'indirizzo IP di WSL (da WSL: `hostname -I`) e collega `http://:4096`.
Quando usi `--hostname 0.0.0.0`, imposta `CLOSEDCODE_SERVER_PASSWORD` per proteggere il server. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Client web + WSL

Per la migliore esperienza web su Windows:

  1. Esegui closedcode web nel terminale WSL invece che in PowerShell:

    closedcode web --hostname 0.0.0.0

  2. Accedi dal browser Windows su http://localhost:<port> (vanilla-closedcode stampa l'URL)

Eseguire closedcode web da WSL garantisce accesso corretto al file system e integrazione del terminale, restando comunque accessibile dal browser Windows.

Accesso ai file Windows

WSL puo accedere a tutti i file Windows tramite la directory /mnt/:

  • Unita C: → /mnt/c/
  • Unita D: → /mnt/d/
  • E cosi via...

Esempio:

cd /mnt/c/Users/YourName/Documents/project
closedcode
Per un'esperienza piu fluida, valuta di clonare o copiare il repository nel file system di WSL (per esempio in `~/code/`) ed eseguire vanilla-closedcode li.
---

Suggerimenti

  • Tieni vanilla-closedcode in esecuzione su WSL per i progetti salvati su unita Windows: l'accesso ai file e immediato
  • Usa l'estensione WSL di VS Code insieme a vanilla-closedcode per un flusso di lavoro integrato
  • Configurazione e sessioni di vanilla-closedcode sono salvate nell'ambiente WSL in ~/.local/share/closedcode/
