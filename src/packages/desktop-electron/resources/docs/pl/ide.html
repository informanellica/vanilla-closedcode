IDE

Rozszerzenie closedcode dla VS Code, Cursor i innych IDE

closedcode integruje się z VS Code, Cursorem lub obsługującym terminal IDE. Aby uruchomić, po prostu uruchom closedcode w terminalu.

Stosowanie

  • Szybkie uruchomienie: użycie Cmd+Esc (Mac) lub Ctrl+Esc (Windows/Linux), aby uruchomić closedcode w uruchamianiu terminala lub skoncentruj się na uruchamianiu terminala, jeśli jest już uruchomiony.
  • Nowa sesja: Cmd+Shift+Esc (Mac) lub Ctrl+Shift+Esc (Windows/Linux), aby została nowa konfiguracja terminala closedcode, nawet jeśli jest już otwarta. Możesz także kliknąć przycisk closedcode w interfejsie użytkownika.
  • Świadomość kontekstu: Automatycznie udostępnia wybór lub kartę za pomocą closedcode.
  • Skróty do odwołania do plików: Cmd+Option+K (Mac) lub Alt+Ctrl+K (Linux/Windows), aby wstawić odnośnik do plików. Na przykład @File#L37-42.

Instalacja

Aby poznać closedcode na VS Code w forkach, takich jak Cursor, Windsurf, VSCodium:

  1. Otwórz kod VS
  2. Otwórz innowacyjny terminal
  3. Uruchom closedcode — rozszerzenie instaluje się automatycznie

Jeśli z drugiej strony chcesz zainstalować własne IDE po uruchomieniu /editor lub /export z TUI, należy zainstalować export EDITOR="code --wait". Dowiedz się więcej.

Instalacja ręczna

Wyszukaj closedcode na rynku rozszerzonym i kliknij Zainstaluj.

Rozwiązywanie problemów

Jeśli rozwiązanie nie zostanie zainstalowane automatycznie:

  • zastosowanie się, że użycie closedcode w terminalu.
  • zadziałanie, że CLI dla Twojego IDE jest zainstalowane:
    • Dla kodu VS: polecenie code
    • Dla kura: polecenie cursor
    • Dla windsurfingu: decyzja windsurf
    • Dla VSCodium: decyzja codium
    • Jeśli nie, uruchom Cmd+Shift+P (Mac) lub Ctrl+Shift+P (Windows/Linux) i wyszukaj „Polecenie: zastosowanie decyzji„ kod ”w PATH” (lub właściwyk dla twojego IDE)
  • zadziałanie, że VS Code ma pozwolenie na instalację rozszerzoną
