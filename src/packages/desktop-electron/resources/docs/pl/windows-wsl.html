Windows (WSL)

Uruchamiaj closedcode na Windows z WSL dla najlepszych efektow.

Chociaz closedcode moze dzialac bezposrednio na Windows, dla najlepszych rezultatow zalecamy Windows Subsystem for Linux (WSL). WSL zapewnia srodowisko Linuksa, ktore bardzo dobrze wspolpracuje z funkcjami closedcode.

Dlaczego WSL?

WSL oferuje lepsza wydajnosc systemu plikow, pelna obsluge terminala i zgodnosc z narzedziami developerskimi, na ktorych opiera sie closedcode.
---

Konfiguracja

  1. Zainstaluj WSL

    Jesli jeszcze tego nie zrobiles, zainstaluj WSL wedlug oficjalnego przewodnika Microsoft.

  2. Zainstaluj closedcode w WSL

    Po skonfigurowaniu WSL otworz terminal WSL i zainstaluj closedcode, pobierajac wydanie ze strony Releases.

  3. Korzystaj z closedcode przez WSL

    Przejdz do katalogu projektu (pliki Windows sa dostepne przez /mnt/c/, /mnt/d/ itd.) i uruchom closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

Aplikacja desktopowa + serwer WSL

Jesli wolisz aplikacje desktopowa closedcode, ale chcesz uruchamiac serwer w WSL:

  1. Uruchom serwer w WSL z --hostname 0.0.0.0, aby zezwolic na polaczenia zewnetrzne:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Polacz aplikacje desktopowa z http://localhost:4096

Jesli `localhost` nie dziala w Twojej konfiguracji, polacz sie przez adres IP WSL (w WSL: `hostname -I`) i uzyj `http://:4096`.
Przy uzyciu `--hostname 0.0.0.0` ustaw `CLOSEDCODE_SERVER_PASSWORD`, aby zabezpieczyc serwer. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Klient web + WSL

Dla najlepszej pracy w przegladarce na Windows:

  1. Uruchamiaj closedcode web w terminalu WSL, a nie w PowerShell:

    closedcode web --hostname 0.0.0.0

  2. Otworz w przegladarce Windows pod adresem http://localhost:<port> (closedcode wypisze URL)

Uruchamianie closedcode web z WSL zapewnia poprawny dostep do systemu plikow i integracje z terminalem, a nadal pozostaje dostepne z przegladarki Windows.

Dostep do plikow Windows

WSL moze uzyskac dostep do wszystkich plikow Windows przez katalog /mnt/:

  • Dysk C: → /mnt/c/
  • Dysk D: → /mnt/d/
  • I tak dalej...

Przyklad:

cd /mnt/c/Users/YourName/Documents/project
closedcode
Aby uzyskac najplynniejsza prace, rozwaz sklonowanie/skopiowanie repozytorium do systemu plikow WSL (np. do `~/code/`) i uruchamianie tam closedcode.
---

Wskazowki

  • Uruchamiaj closedcode w WSL dla projektow zapisanych na dyskach Windows - dostep do plikow jest bezproblemowy
  • Uzywaj rozszerzenia WSL dla VS Code razem z closedcode, aby miec zintegrowany workflow
  • Konfiguracja closedcode i sesje sa przechowywane w srodowisku WSL pod ~/.local/share/closedcode/
