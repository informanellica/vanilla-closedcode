Windows (WSL)

شغّل vanilla-closedcode على Windows باستخدام WSL لأفضل تجربة.

رغم أن vanilla-closedcode يمكن تشغيله مباشرة على Windows، نوصي باستخدام Windows Subsystem for Linux (WSL) للحصول على أفضل تجربة. يوفر WSL بيئة Linux تعمل بسلاسة مع ميزات vanilla-closedcode.

لماذا WSL؟

يوفر WSL أداء أفضل لنظام الملفات، ودعمًا كاملًا لـ terminal، وتوافقًا مع أدوات التطوير التي يعتمد عليها vanilla-closedcode.
---

الإعداد

  1. ثبّت WSL

    إذا لم تقم بذلك بعد، ثبّت WSL باستخدام دليل Microsoft الرسمي.

  2. ثبّت vanilla-closedcode داخل WSL

    بعد إعداد WSL، افتح terminal داخل WSL وثبّت vanilla-closedcode باستخدام إحدى طرق التثبيت.

    نزّل أحدث إصدار من صفحة الإصدارات: github.com/informanellica/vanilla-closedcode/releases.

  3. استخدم vanilla-closedcode من WSL

    انتقل إلى مجلد مشروعك (يمكنك الوصول لملفات Windows عبر /mnt/c/ و/mnt/d/ وغيرها) ثم شغّل vanilla-closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

تطبيق سطح المكتب + خادم WSL

إذا كنت تفضّل تطبيق vanilla-closedcode لسطح المكتب لكن تريد تشغيل الخادم داخل WSL:

  1. ابدأ الخادم داخل WSL مع --hostname 0.0.0.0 للسماح بالاتصالات الخارجية:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. اربط تطبيق سطح المكتب على http://localhost:4096

إذا لم يعمل `localhost` في إعدادك، اتصل باستخدام عنوان IP الخاص بـ WSL بدلًا منه (من داخل WSL: `hostname -I`) واستخدم `http://:4096`.
عند استخدام `--hostname 0.0.0.0`، اضبط `CLOSEDCODE_SERVER_PASSWORD` لحماية الخادم. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

عميل الويب + WSL

لأفضل تجربة ويب على Windows:

  1. شغّل closedcode web من terminal داخل WSL بدلًا من PowerShell:

    closedcode web --hostname 0.0.0.0

  2. افتحه من متصفح Windows عبر http://localhost:<port> (يعرض vanilla-closedcode الرابط)

تشغيل closedcode web من WSL يضمن وصولًا صحيحًا لنظام الملفات وتكاملًا أفضل مع terminal، مع بقائه متاحًا من متصفح Windows.

الوصول إلى ملفات Windows

يمكن لـ WSL الوصول إلى جميع ملفات Windows عبر مجلد /mnt/:

  • قرص C: → /mnt/c/
  • قرص D: → /mnt/d/
  • وهكذا...

مثال:

cd /mnt/c/Users/YourName/Documents/project
closedcode
لأفضل سلاسة، يمكنك استنساخ/نسخ المستودع إلى نظام ملفات WSL (مثل `~/code/`) وتشغيل vanilla-closedcode من هناك.
---

نصائح

  • شغّل vanilla-closedcode داخل WSL للمشاريع المخزنة على أقراص Windows حتى يكون الوصول للملفات سلسًا
  • استخدم إضافة WSL في VS Code مع vanilla-closedcode لسير عمل تطوير متكامل
  • إعدادات vanilla-closedcode وجلساته تُخزَّن داخل بيئة WSL في ~/.local/share/closedcode/
