CLI

أوامر وخيارات CLI في vanilla-closedcode.

يبدأ CLI الخاص بـ vanilla-closedcode افتراضيا واجهة TUI عند تشغيله بدون أي معاملات.

closedcode

كما يدعم أيضا أوامر موثقة في هذه الصفحة، ما يتيح لك التفاعل مع vanilla-closedcode برمجيا.

closedcode run "Explain how closures work in JavaScript"

tui

بدء واجهة المستخدم terminal (TUI) الخاصة بـ vanilla-closedcode.

closedcode [project]

الخيارات

الخيار المختصر الوصف
{"--continue"} -c متابعة الجلسة الأخيرة
{"--session"} -s معرّف الجلسة للمتابعة
{"--fork"} تفريع الجلسة عند المتابعة (يستخدم مع --continue أو --session)
{"--prompt"} الموجّه المراد استخدامه
{"--model"} -m النموذج المراد استخدامه بصيغة provider/model
{"--agent"} الوكيل المراد استخدامه
{"--port"} المنفذ الذي يتم الاستماع عليه
{"--hostname"} اسم المضيف الذي يتم الاستماع عليه

الأوامر

يتضمن CLI الخاص بـ vanilla-closedcode أيضا الأوامر التالية.

agent

إدارة وكلاء vanilla-closedcode.

closedcode agent [command]

attach

إرفاق terminal بخادم الواجهة الخلفية لـ vanilla-closedcode قيد التشغيل بالفعل، والذي تم تشغيله عبر الأمرين serve أو web.

closedcode attach [url]

يتيح ذلك استخدام واجهة TUI مع واجهة خلفية لـ vanilla-closedcode تعمل عن بعد. على سبيل المثال:

# ابدأ خادم الواجهة الخلفية للوصول عبر الويب/الجوال
closedcode web --port 4096 --hostname 0.0.0.0

# في محطة طرفية (terminal) أخرى، اربط TUI بالواجهة الخلفية قيد التشغيل
closedcode attach http://10.20.30.40:4096

الرايات

الراية المختصر الوصف
{"--dir"} دليل العمل الذي ستبدأ منه واجهة TUI
{"--continue"} -c متابعة آخر جلسة
{"--session"} -s معرّف الجلسة للمتابعة
{"--fork"} تفريع الجلسة عند المتابعة (استخدمه مع --continue أو --session)
{"--password"} -p كلمة مرور المصادقة الأساسية (الافتراضي CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u اسم مستخدم المصادقة الأساسية (الافتراضي CLOSEDCODE_SERVER_USERNAME أو closedcode)

create

إنشاء وكيل جديد بإعدادات مخصصة.

closedcode agent create

سيرشدك هذا الأمر خلال إنشاء وكيل جديد مع موجه نظام مخصص وإعدادات الأدوات.

list

عرض جميع الوكلاء المتاحين.

closedcode agent list

auth

أمر لإدارة بيانات الاعتماد وتسجيل الدخول لمزودي الخدمة.

closedcode auth [command]

login

يعتمد vanilla-closedcode على قائمة المزودين في Models.dev، لذا يمكنك استخدام closedcode auth login لتهيئة مفاتيح API لأي مزود ترغب باستخدامه. يتم حفظ ذلك في ~/.local/share/closedcode/auth.json.

closedcode auth login

عند تشغيل vanilla-closedcode يقوم بتحميل المزودين من ملف بيانات الاعتماد، وكذلك أي مفاتيح معرّفة في متغيرات البيئة لديك أو في ملف .env ضمن مشروعك.

list

يسرد جميع المزودين الذين تم توثيقهم كما هم محفوظون في ملف بيانات الاعتماد.

closedcode auth list

أو النسخة المختصرة.

closedcode auth ls

logout

يسجلك خارج مزود عبر حذفه من ملف بيانات الاعتماد.

closedcode auth logout

github

إدارة وكيل GitHub لأتمتة المستودع.

closedcode github [command]

install

تثبيت وكيل GitHub في مستودعك.

closedcode github install

يقوم ذلك بإعداد سير عمل GitHub Actions اللازم ويرشدك خلال عملية التهيئة. اعرف المزيد.

run

تشغيل وكيل GitHub. يُستخدم هذا عادة ضمن GitHub Actions.

closedcode github run
الرايات
الراية الوصف
{"--event"} حدث GitHub مُحاكى لتشغيل الوكيل عليه
{"--token"} رمز وصول شخصي لـ GitHub

mcp

إدارة خوادم Model Context Protocol.

closedcode mcp [command]

add

إضافة خادم MCP إلى تهيئتك.

closedcode mcp add

سيرشدك هذا الأمر خلال إضافة خادم MCP محلي أو بعيد.

list

عرض جميع خوادم MCP المُهيأة وحالة اتصالها.

closedcode mcp list

أو استخدم النسخة المختصرة.

closedcode mcp ls

auth

إجراء المصادقة مع خادم MCP يدعم OAuth.

closedcode mcp auth [name]

إذا لم تُحدِّد اسم خادم، فسيُطلب منك الاختيار من الخوادم المتاحة الداعمة لـ OAuth.

يمكنك أيضا عرض الخوادم الداعمة لـ OAuth وحالة المصادقة الخاصة بها.

closedcode mcp auth list

أو استخدم النسخة المختصرة.

closedcode mcp auth ls

logout

إزالة بيانات اعتماد OAuth لخادم MCP.

closedcode mcp logout [name]

debug

تشخيص مشاكل اتصال OAuth لخادم MCP.

closedcode mcp debug <name>

models

عرض جميع النماذج المتاحة من المزودين المُهيأين.

closedcode models [provider]

يعرض هذا الأمر جميع النماذج المتاحة عبر المزودين المُهيأين لديك بصيغة provider/model.

هذا مفيد لتحديد اسم النموذج الدقيق الذي ستستخدمه في تهيئتك.

يمكنك اختياريا تمرير معرّف مزود لتصفية النماذج حسب ذلك المزود.

closedcode models anthropic

الرايات

الراية الوصف
{"--refresh"} تحديث ذاكرة التخزين المؤقت للنماذج من models.dev
{"--verbose"} استخدام مخرجات أكثر تفصيلا للنماذج (تشمل بيانات مثل التكاليف)

استخدم الراية --refresh لتحديث قائمة النماذج المخزنة مؤقتا. يفيد ذلك عند إضافة نماذج جديدة إلى مزود وتريد رؤيتها في vanilla-closedcode.

closedcode models --refresh

run

تشغيل closedcode في وضع غير تفاعلي عبر تمرير موجه مباشرة.

closedcode run [message..]

هذا مفيد للبرمجة النصية والأتمتة، أو عندما تريد إجابة سريعة دون تشغيل واجهة TUI كاملة. على سبيل المثال.

closedcode run Explain the use of context in Go

يمكنك أيضا الإرفاق بمثيل closedcode serve قيد التشغيل لتجنّب زمن الإقلاع البارد لخوادم MCP في كل تشغيل:

# ابدأ خادمًا بلا واجهة في محطة طرفية واحدة
closedcode serve

# في محطة طرفية أخرى، شغّل الأوامر التي ترتبط به
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

الراية المختصر الوصف
{"--command"} الأمر المراد تشغيله؛ استخدم الرسالة كوسائط
{"--continue"} -c متابعة الجلسة الأخيرة
{"--session"} -s معرّف الجلسة للمتابعة

serve

بدء خادم vanilla-closedcode بدون واجهة للوصول عبر API. راجع توثيق الخادم لواجهة HTTP الكاملة.

closedcode serve

يشغّل هذا خادم HTTP يوفّر وصولا عبر API لوظائف closedcode دون واجهة TUI. اضبط CLOSEDCODE_SERVER_PASSWORD لتفعيل مصادقة HTTP الأساسية (اسم المستخدم الافتراضي closedcode).

الرايات

الراية الوصف
{"--port"} المنفذ الذي يتم الاستماع عليه
{"--hostname"} اسم المضيف الذي يتم الاستماع عليه
{"--mdns"} تفعيل اكتشاف mDNS
{"--cors"} أصول/منشأات إضافية للمتصفح للسماح بـ CORS
