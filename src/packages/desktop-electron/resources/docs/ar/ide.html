IDE

امتداد vanilla-closedcode لـ VS Code و Cursor وغيرها من بيئات التطوير

يتكامل vanilla-closedcode مع VS Code و Cursor أو أي IDE يدعم terminal. ما عليك سوى تشغيل closedcode في terminal للبدء.

الاستخدام

  • تشغيل سريع: استخدم Cmd+Esc (Mac) أو Ctrl+Esc (Windows/Linux) لفتح vanilla-closedcode في عرض terminal مقسّم، أو للتركيز على جلسة terminal موجودة إذا كانت قيد التشغيل بالفعل.
  • جلسة جديدة: استخدم Cmd+Shift+Esc (Mac) أو Ctrl+Shift+Esc (Windows/Linux) لبدء جلسة terminal جديدة لـ vanilla-closedcode حتى لو كانت هناك جلسة مفتوحة. يمكنك أيضا النقر على زر vanilla-closedcode في واجهة المستخدم.
  • وعي بالسياق: شارك تلقائيا تحديدك الحالي أو تبويبك مع vanilla-closedcode.
  • اختصارات الإشارة إلى الملفات: استخدم Cmd+Option+K (Mac) أو Alt+Ctrl+K (Linux/Windows) لإدراج مراجع الملفات. مثلا: @File#L37-42.

التثبيت

لتثبيت vanilla-closedcode على VS Code والتفرعات الشائعة مثل Cursor و Windsurf و VSCodium:

  1. افتح VS Code
  2. افتح terminal المدمجة
  3. شغّل closedcode - سيتم تثبيت الامتداد تلقائيا

أما إذا كنت تريد استخدام IDE الخاص بك عند تشغيل /editor أو /export من واجهة TUI، فستحتاج إلى ضبط export EDITOR="code --wait". اعرف المزيد.

التثبيت اليدوي

ابحث عن vanilla-closedcode في Extension Marketplace وانقر Install.

استكشاف الأخطاء وإصلاحها

إذا فشل تثبيت الامتداد تلقائيا:

  • تأكد من أنك تشغّل closedcode داخل terminal المدمجة.
  • تأكد من تثبيت CLI الخاصة بـ IDE لديك:
    • لـ VS Code: أمر code
    • لـ Cursor: أمر cursor
    • لـ Windsurf: أمر windsurf
    • لـ VSCodium: أمر codium
    • إذا لم تكن مثبتة، شغّل Cmd+Shift+P (Mac) أو Ctrl+Shift+P (Windows/Linux) وابحث عن "Shell Command: Install 'code' command in PATH" (أو ما يعادله في IDE لديك)
  • تأكد من أن VS Code لديه الصلاحية لتثبيت الامتدادات
