الأدوات المخصصة

أنشئ أدوات يمكن لـ LLM استدعاؤها في closedcode.

الأدوات المخصصة هي دوال تنشئها يمكن لـ LLM استدعاؤها أثناء المحادثات. تعمل جنبا إلى جنب مع الأدوات المدمجة في closedcode مثل read وwrite وbash.

إنشاء أداة

تُعرّف الأدوات كملفات TypeScript أو JavaScript. لكن تعريف الأداة يمكنه استدعاء سكربتات مكتوبة بأي لغة — إذ تُستخدم TypeScript أو JavaScript فقط لتعريف الأداة نفسه.

الموقع

يمكن تعريفها:

  • محليا بوضعها في مجلد .closedcode/tools/ داخل مشروعك.
  • أو عالميا بوضعها في ~/.config/closedcode/tools/.

البنية

أسهل طريقة لإنشاء أدوات هي استخدام المساعد tool() الذي يوفر أمان الأنواع والتحقق.




  description: "Query the project database",
  args: {
    query: tool.schema.string().describe("SQL query to execute"),
  },
  async execute(args) {
    // Your database logic here
    return `Executed query: ${args.query}`
  },
})

يصبح اسم الملف هو اسم الأداة. المثال أعلاه ينشئ أداة باسم database.

عدة أدوات في ملف واحد

يمكنك أيضا تصدير عدة أدوات من ملف واحد. يصبح كل تصدير أداة مستقلة باسم <filename>_<exportname>:




  description: "Add two numbers",
  args: {
    a: tool.schema.number().describe("First number"),
    b: tool.schema.number().describe("Second number"),
  },
  async execute(args) {
    return args.a + args.b
  },
})


  description: "Multiply two numbers",
  args: {
    a: tool.schema.number().describe("First number"),
    b: tool.schema.number().describe("Second number"),
  },
  async execute(args) {
    return args.a * args.b
  },
})

ينشئ هذا أداتين: math_add وmath_multiply.

تضارب الأسماء مع الأدوات المدمجة

تُصنّف الأدوات المخصصة حسب اسم الأداة. إذا استخدمت أداة مخصصة نفس اسم أداة مدمجة، فإن الأداة المخصصة تأخذ الأولوية.

على سبيل المثال، يستبدل هذا الملف أداة bash المدمجة:




  description: "Restricted bash wrapper",
  args: {
    command: tool.schema.string(),
  },
  async execute(args) {
    return `blocked: ${args.command}`
  },
})
فضّل استخدام أسماء فريدة ما لم تكن تريد استبدال أداة مدمجة عمدا. إذا كنت تريد تعطيل أداة مدمجة ولكن لا تريد استبدالها، استخدم [الأذونات](/docs/permissions).
---

الوسائط

يمكنك استخدام tool.schema، وهو في الأساس Zod، لتعريف أنواع الوسائط.

args: {
  query: tool.schema.string().describe("SQL query to execute")
}

يمكنك أيضا استيراد Zod مباشرة وإرجاع كائن عادي:




  description: "Tool description",
  args: {
    param: z.string().describe("Parameter description"),
  },
  async execute(args, context) {
    // Tool implementation
    return "result"
  },
}

السياق

تستقبل الأدوات سياقا حول الجلسة الحالية:




  description: "Get project information",
  args: {},
  async execute(args, context) {
    // Access context information
    const { agent, sessionID, messageID, directory, worktree } = context
    return `Agent: ${agent}, Session: ${sessionID}, Message: ${messageID}, Directory: ${directory}, Worktree: ${worktree}`
  },
})

استخدم context.directory لدليل العمل الخاص بالجلسة. استخدم context.worktree لجذر شجرة العمل (worktree) في git.

أمثلة

كتابة أداة بلغة Python

يمكنك كتابة أدواتك بأي لغة تريدها. إليك مثالا يجمع رقمين باستخدام Python.

أولا، أنشئ الأداة كسكربت Python:



a = int(sys.argv[1])
b = int(sys.argv[2])
print(a + b)

ثم أنشئ تعريف الأداة الذي يستدعيه:


import { execSync } from "node:child_process"
import path from "node:path"

  description: "Add two numbers using Python",
  args: {
    a: tool.schema.number().describe("First number"),
    b: tool.schema.number().describe("Second number"),
  },
  async execute(args, context) {
    const script = path.join(context.worktree, ".closedcode/tools/add.py")
    const result = execSync(`python3 ${script} ${args.a} ${args.b}`, { encoding: "utf8" })
    return result.trim()
  },
})

هنا نستخدم أداة child_process لتشغيل سكربت Python.
