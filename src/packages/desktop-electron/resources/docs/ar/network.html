الشبكة

إعداد الوكلاء والشهادات المخصصة.

يدعم vanilla-closedcode متغيرات بيئة الوكيل (proxy) القياسية والشهادات المخصصة لبيئات الشبكات المؤسسية.

الوكيل

يتبع vanilla-closedcode متغيرات بيئة الوكيل القياسية.

# وكيل HTTPS (موصى به)


# وكيل HTTP (إذا لم يتوفر HTTPS)


# تجاوز الوكيل للخادم المحلي (مطلوب)
تتواصل واجهة TUI مع خادم HTTP محلي. يجب تجاوز الوكيل لهذا الاتصال لمنع حلقات التوجيه.
يمكنك إعداد منفذ الخادم واسم المضيف باستخدام [أعلام CLI](/docs/cli#run).

المصادقة

إذا كان الوكيل يتطلب مصادقة أساسية، فأدرج بيانات الاعتماد في عنوان URL.
تجنّب تضمين كلمات المرور بشكل ثابت. استخدم متغيرات البيئة أو تخزين بيانات اعتماد آمن.
بالنسبة للوكلاء الذين يتطلبون مصادقة متقدمة مثل NTLM أو Kerberos، فكّر في استخدام بوابة LLM تدعم طريقة المصادقة لديك.

الشهادات المخصصة

إذا كانت مؤسستك تستخدم سلطات تصديق (CA) مخصصة لاتصالات HTTPS، فقم بإعداد vanilla-closedcode ليثق بها.

يعمل ذلك لكل من اتصالات الوكيل والوصول المباشر إلى APIs.
