CLI

Options et commandes de la CLI vanilla-closedcode.

La CLI vanilla-closedcode démarre par défaut le TUI lorsqu'il est exécuté sans aucun argument.

closedcode

Mais elle accepte également les commandes documentées sur cette page. Cela vous permet d'interagir avec vanilla-closedcode par programme.

closedcode run "Explain how closures work in JavaScript"

tui

Démarrez l’interface utilisateur du terminal vanilla-closedcode.

closedcode [project]

Options

Option Court Description
{"--continue"} -c Continuer la dernière session
{"--session"} -s ID de session pour continuer
{"--fork"} Forker la session en continuant (à utiliser avec --continue ou --session)
{"--prompt"} Prompt à utiliser
{"--model"} -m Modèle à utiliser sous forme de fournisseur/modèle
{"--agent"} Agent à utiliser
{"--port"} Port d'écoute
{"--hostname"} Nom d'hôte d'écoute

Commandes

La CLI vanilla-closedcode dispose également des commandes suivantes.

agent

Gérer les agents pour vanilla-closedcode.

closedcode agent [command]

attach

Attachez un terminal à un serveur backend vanilla-closedcode déjà en cours d'exécution démarré via les commandes serve ou web.

closedcode attach [url]

Cela permet d'utiliser le TUI avec un backend vanilla-closedcode distant. Par exemple :

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Options

Option Court Description
{"--dir"} Répertoire de travail dans lequel démarrer TUI
{"--continue"} -c Continuer la dernière session
{"--session"} -s ID de session pour continuer
{"--fork"} Dupliquer la session lors de la reprise (à utiliser avec --continue ou --session)
{"--password"} -p Mot de passe d'authentification de base (par défaut CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Nom d'utilisateur d'authentification de base (par défaut CLOSEDCODE_SERVER_USERNAME ou closedcode)

create

Créez un nouvel agent avec une configuration personnalisée.

closedcode agent create

Cette commande vous guidera dans la création d'un nouvel agent avec un prompt système personnalisé et une configuration d'outils.

list

Liste tous les agents disponibles.

closedcode agent list

auth

Commande pour gérer les informations d’identification et la connexion des fournisseurs.

closedcode auth [command]

login

vanilla-closedcode est alimenté par la liste des fournisseurs sur Models.dev, vous pouvez donc utiliser closedcode auth login pour configurer les clés API pour tout fournisseur que vous souhaitez utiliser. Ceci est stocké dans ~/.local/share/closedcode/auth.json.

closedcode auth login

Lorsque vanilla-closedcode démarre, il charge les fournisseurs à partir du fichier d'informations d'identification, ainsi que les clés définies dans vos environnements ou un fichier .env dans votre projet.

list

Liste tous les fournisseurs authentifiés tels qu'ils sont stockés dans le fichier d'informations d'identification.

closedcode auth list

Ou la version courte.

closedcode auth ls

logout

Vous déconnecte d'un fournisseur en l'effaçant du fichier d'informations d'identification.

closedcode auth logout

github

Gérez l'agent GitHub pour l'automatisation du dépôt.

closedcode github [command]

install

Installez l'agent GitHub dans votre dépôt.

closedcode github install

Cela configure le flux de travail GitHub Actions nécessaire et vous guide tout au long du processus de configuration. En savoir plus.

run

Exécutez l'agent GitHub. Ceci est généralement utilisé dans les GitHub Actions.

closedcode github run
Options
Option Description
{"--event"} Événement simulé GitHub pour lequel exécuter l'agent
{"--token"} Jeton d'accès personnel GitHub

MCP

Gérer les serveurs Model Context Protocol.

closedcode mcp [command]

add

Ajoutez un serveur MCP à votre configuration.

closedcode mcp add

Cette commande vous guidera dans l’ajout d’un serveur MCP local ou distant.

list

Liste tous les serveurs MCP configurés et leur état de connexion.

closedcode mcp list

Ou utilisez la version courte.

closedcode mcp ls

auth

Authentifiez-vous auprès d'un serveur MCP compatible OAuth.

closedcode mcp auth [name]

Si vous ne fournissez pas de nom de serveur, vous serez invité à sélectionner parmi les serveurs compatibles OAuth disponibles.

Vous pouvez également répertorier les serveurs compatibles OAuth et leur statut d'authentification.

closedcode mcp auth list

Ou utilisez la version courte.

closedcode mcp auth ls

logout

Supprimez les informations d'identification OAuth pour un serveur MCP.

closedcode mcp logout [name]

debug

Déboguer les problèmes de connexion OAuth pour un serveur MCP.

closedcode mcp debug <name>

models

Liste tous les modèles disponibles auprès des fournisseurs configurés.

closedcode models [provider]

Cette commande affiche tous les modèles disponibles parmi vos fournisseurs configurés au format provider/model.

Ceci est utile pour déterminer le nom exact du modèle à utiliser dans votre config.

Vous pouvez éventuellement transmettre un ID de fournisseur pour filtrer les modèles en fonction de ce fournisseur.

closedcode models anthropic

Options

Option Description
{"--refresh"} Actualisez le cache des modèles à partir de models.dev
{"--verbose"} Utiliser une sortie de modèle plus détaillée (inclut des métadonnées telles que les coûts)

Utilisez l'option --refresh pour mettre à jour la liste des modèles mis en cache. Ceci est utile lorsque de nouveaux modèles ont été ajoutés à un fournisseur et que vous souhaitez les voir dans vanilla-closedcode.

closedcode models --refresh

run

Exécutez closedcode en mode non interactif en transmettant directement un prompt.

closedcode run [message..]

Ceci est utile pour les scripts, l'automatisation ou lorsque vous souhaitez une réponse rapide sans lancer le TUI complet. Par exemple :

closedcode run Explain the use of context in Go

Vous pouvez également vous connecter à une instance closedcode serve en cours d'exécution pour éviter les temps de démarrage à froid du serveur MCP à chaque exécution :

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Options

Option Court Description
{"--command"} La commande à exécuter, utilisez le message pour les arguments
{"--continue"} -c Continuer la dernière session
{"--session"} -s ID de session pour continuer
{"--fork"} Forker la session en continuant (à utiliser avec --continue ou --session)
{"--share"} Partager la session
{"--model"} -m Modèle à utiliser sous forme de fournisseur/modèle
{"--agent"} Agent à utiliser
{"--file"} -f Fichier(s) à joindre au message
{"--format"} Format : par défaut (formaté) ou json (événements JSON bruts)
{"--title"} Titre de la session (utilise un prompt tronqué si aucune valeur n'est fournie)
{"--attach"} Connectez-vous à un serveur closedcode en cours d'exécution (par exemple, http://localhost:4096)
{"--password"} -p Mot de passe d'authentification de base (par défaut CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Nom d'utilisateur d'authentification de base (par défaut CLOSEDCODE_SERVER_USERNAME ou closedcode)
{"--dir"} Répertoire d'exécution, ou chemin sur le serveur distant lors de l'attachement
{"--variant"} Variante du modèle (effort de raisonnement spécifique au fournisseur)
{"--thinking"} Afficher les blocs de réflexion
{"--port"} Port du serveur local (port aléatoire par défaut)

serve

Démarrez un serveur vanilla-closedcode sans interface graphique pour un accès API. Consultez le serveur docs pour l'interface complète HTTP.

closedcode serve

Cela démarre un serveur HTTP qui fournit un accès API aux fonctionnalités d'vanilla-closedcode sans l'interface TUI. Définissez CLOSEDCODE_SERVER_PASSWORD pour activer l'authentification de base HTTP (le nom d'utilisateur par défaut est closedcode).

Options

Option Description
{"--port"} Port d'écoute
{"--hostname"} Nom d'hôte d'écoute
{"--mdns"} Activer la découverte mDNS
{"--cors"} Origines de navigateur supplémentaires pour autoriser CORS

session

Gérer les sessions vanilla-closedcode.

closedcode session [command]

list

Liste toutes les sessions vanilla-closedcode.

closedcode session list
Options
Option Court Description
{"--max-count"} -n Limiter aux N sessions les plus récentes
{"--format"} Format de sortie : tableau ou json (tableau)

stats

Affichez les statistiques d'utilisation et de coût des jetons pour vos sessions vanilla-closedcode.

closedcode stats

Options

Option Description
{"--days"} Afficher les statistiques des N derniers jours (depuis le début)
{"--tools"} Nombre d'outils à afficher (tous)
{"--models"} Afficher la répartition de l'utilisation du modèle (masqué par défaut). Passez un numéro pour afficher les N premiers
{"--project"} Filtrer par projet (tous les projets, chaîne vide : projet actuel)

export

Exportez les données de session au format JSON.

closedcode export [sessionID]

Si vous ne fournissez pas d'ID de session, vous serez invité à sélectionner parmi les sessions disponibles.

import

Importez les données de session à partir d'un fichier JSON ou d'une URL de partage vanilla-closedcode.

closedcode import <file>

Vous pouvez importer à partir d'un fichier local ou d'une URL de partage vanilla-closedcode.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

Démarrez un serveur vanilla-closedcode sans interface graphique avec une interface Web.

closedcode web

Cela démarre un serveur HTTP et ouvre un navigateur Web pour accéder à vanilla-closedcode via une interface Web. Définissez CLOSEDCODE_SERVER_PASSWORD pour activer l'authentification de base HTTP (le nom d'utilisateur par défaut est closedcode).

Options

Option Description
{"--port"} Port d'écoute
{"--hostname"} Nom d'hôte d'écoute
{"--mdns"} Activer la découverte mDNS
{"--cors"} Origines de navigateur supplémentaires pour autoriser CORS

acp

Démarrez un serveur ACP (Agent Client Protocol).

closedcode acp

Cette commande démarre un serveur ACP qui communique via stdin/stdout en utilisant nd-JSON.

Options

Option Description
{"--cwd"} Répertoire de travail
{"--port"} Port d'écoute
{"--hostname"} Nom d'hôte d'écoute

uninstall

Désinstallez vanilla-closedcode et supprimez tous les fichiers associés.

closedcode uninstall

Options

Option Court Description
{"--keep-config"} -c Conserver les fichiers de configuration
{"--keep-data"} -d Conserver les données de session et les instantanés
{"--dry-run"} Afficher ce qui serait supprimé sans supprimer
{"--force"} -f Ignorer les invites de confirmation

upgrade

Met à jour closedcode vers la dernière version ou une version spécifique.

closedcode upgrade [target]

Pour mettre à niveau vers la dernière version.

closedcode upgrade

Pour passer à une version spécifique.

closedcode upgrade v0.1.48

Options

Option Court Description
{"--method"} -m La méthode d'installation utilisée ; curl, npm, pnpm, brew

Variables d'environnement

vanilla-closedcode peut être configuré à l'aide de variables d'environnement.

Variables Type Description
CLOSEDCODE_AUTO_SHARE booléen Partager automatiquement des sessions
CLOSEDCODE_GIT_BASH_PATH chaîne Chemin vers l'exécutable Git Bash sur Windows
CLOSEDCODE_CONFIG chaîne Chemin d'accès au fichier de configuration
CLOSEDCODE_TUI_CONFIG chaîne Chemin d'accès au fichier de configuration TUI
CLOSEDCODE_CONFIG_DIR chaîne Chemin d'accès au répertoire de configuration
CLOSEDCODE_CONFIG_CONTENT chaîne Contenu de configuration JSON en ligne
CLOSEDCODE_DISABLE_AUTOUPDATE booléen Désactiver les vérifications automatiques des mises à jour
CLOSEDCODE_DISABLE_PRUNE booléen Désactiver le nettoyage des anciennes données
CLOSEDCODE_DISABLE_TERMINAL_TITLE booléen Désactiver les mises à jour automatiques des titres du terminal
CLOSEDCODE_PERMISSION chaîne Configuration des autorisations JSON intégrées
CLOSEDCODE_DISABLE_DEFAULT_PLUGINS booléen Désactiver les plugins par défaut
CLOSEDCODE_DISABLE_LSP_DOWNLOAD booléen Désactiver les téléchargements automatiques du serveur LSP
CLOSEDCODE_ENABLE_EXPERIMENTAL_MODELS booléen Activer les modèles expérimentaux
CLOSEDCODE_DISABLE_AUTOCOMPACT booléen Désactiver le compactage automatique du contexte
CLOSEDCODE_DISABLE_CLAUDE_CODE booléen Désactiver la lecture de .claude (prompt + compétences)
CLOSEDCODE_DISABLE_CLAUDE_CODE_PROMPT booléen Désactiver la lecture ~/.claude/CLAUDE.md
CLOSEDCODE_DISABLE_CLAUDE_CODE_SKILLS booléen Désactiver le chargement de .claude/skills
CLOSEDCODE_DISABLE_MODELS_FETCH booléen Désactiver la récupération de modèles à partir de sources distantes
CLOSEDCODE_FAKE_VCS chaîne Faux fournisseur VCS à des fins de test
CLOSEDCODE_CLIENT chaîne Identifiant du client (par défaut cli)
CLOSEDCODE_ENABLE_EXA booléen Activer les outils de recherche Web Exa
CLOSEDCODE_SERVER_PASSWORD chaîne Activer l'authentification de base pour serve/web
CLOSEDCODE_SERVER_USERNAME chaîne Remplacer le nom d'utilisateur d'authentification de base (par défaut closedcode)
CLOSEDCODE_MODELS_URL chaîne URL personnalisé pour récupérer la configuration des modèles

Variables Type Description

CLOSEDCODE_EXPERIMENTAL_DISABLE_COPY_ON_SELECT booléen Désactiver la copie lors de la sélection dans TUI
CLOSEDCODE_EXPERIMENTAL_BASH_DEFAULT_TIMEOUT_MS numéro Délai d'expiration par défaut pour les commandes bash en ms
CLOSEDCODE_EXPERIMENTAL_OUTPUT_TOKEN_MAX numéro Nombre maximum de jetons de sortie pour les réponses LLM
CLOSEDCODE_EXPERIMENTAL_FILEWATCHER booléen Activer l'observateur de fichiers pour l'ensemble du répertoire
CLOSEDCODE_EXPERIMENTAL_OXFMT booléen Activer le formateur oxfmt
CLOSEDCODE_EXPERIMENTAL_LSP_TOOL booléen Activer l'outil expérimental LSP
CLOSEDCODE_EXPERIMENTAL_DISABLE_FILEWATCHER booléen Désactiver l'observateur de fichiers
CLOSEDCODE_EXPERIMENTAL_EXA booléen Activer les fonctionnalités Exa expérimentales
CLOSEDCODE_EXPERIMENTAL_LSP_TY booléen Activer TY LSP pour les fichiers python
CLOSEDCODE_EXPERIMENTAL_MARKDOWN booléen Activer les fonctionnalités Markdown expérimentales
CLOSEDCODE_EXPERIMENTAL_PLAN_MODE booléen Activer le mode plan
