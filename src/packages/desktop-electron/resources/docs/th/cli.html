CLI

สัญญาณและคำสั่งของ vanilla-closedcode CLI

vanilla-closedcode CLI จะเริ่มต้น TUI เป็นค่าเริ่มต้นเมื่อเรียกใช้โดยไม่มีอาร์กิวเมนต์

closedcode

นอกจากนี้ยังรับคำสั่งและแฟล็ก ซึ่งช่วยให้คุณใช้ vanilla-closedcode ผ่านโปรแกรมได้

closedcode run "Explain how closures work in JavaScript"

tui

เริ่ม TUI เพื่อใช้ vanilla-closedcode แบบโต้ตอบ

closedcode [project]

แฟล็ก

แฟล็ก สั้น คำอธิบาย
{"--continue"} -c ดำเนินการต่อจากเซสชันล่าสุด
{"--session"} -s ID เซสชันเพื่อดำเนินการต่อ
{"--fork"} แยกเซสชันเมื่อทำต่อ (ใช้กับ --continue หรือ --session)
{"--prompt"} พรอมต์เริ่มต้นที่จะใช้
{"--model"} -m โมเดลที่จะใช้ในรูปแบบ provider/model
{"--agent"} เอเจนต์ที่จะใช้
{"--port"} พอร์ตที่จะฟัง
{"--hostname"} ชื่อโฮสต์ที่จะฟัง

คำสั่ง

vanilla-closedcode CLI ยังมีคำสั่งย่อยอีกมากมาย

agent

จัดการเอเจนต์สำหรับ vanilla-closedcode

closedcode agent [command]

attach

เชื่อมต่อกับเซิร์ฟเวอร์แบ็กเอนด์ vanilla-closedcode ที่กำลังทำงานอยู่ ซึ่งเริ่มต้นผ่านคำสั่ง serve หรือ web

closedcode attach [url]

วิธีนี้ช่วยให้คุณใช้ TUI กับแบ็กเอนด์ vanilla-closedcode ระยะไกลได้ ตัวอย่าง:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

แฟล็ก

แฟล็ก สั้น คำอธิบาย
{"--dir"} ไดเร็กทอรีการทำงานเพื่อเริ่มต้น TUI ใน
{"--continue"} -c ดำเนินการต่อจากเซสชันล่าสุด
{"--session"} -s ID เซสชันเพื่อดำเนินการต่อ
{"--fork"} แยกเซสชันเมื่อดำเนินการต่อ (ใช้กับ --continue หรือ --session)
{"--password"} -p รหัสผ่านการยืนยันตัวตนพื้นฐาน (ค่าเริ่มต้นคือ CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u ชื่อผู้ใช้การยืนยันตัวตนพื้นฐาน (ค่าเริ่มต้นคือ CLOSEDCODE_SERVER_USERNAME หรือ closedcode)

create

สร้างเอเจนต์ใหม่

closedcode agent create

คำสั่งนี้จะแนะนำคุณตลอดกระบวนการสร้างเอเจนต์ใหม่ พร้อมพรอมต์ระบบและความสามารถของเครื่องมือ

list

แสดงรายชื่อเอเจนต์ที่พร้อมใช้งานทั้งหมด

closedcode agent list

auth

จัดการการตรวจสอบสิทธิ์และข้อมูลรับรอง

closedcode auth [command]

login

vanilla-closedcode ได้รับการกำหนดค่าไว้ล่วงหน้าด้วยรายชื่อผู้ให้บริการทั่วไปที่ Models.dev ดังนั้นคุณเพียงแค่ต้องใส่คีย์ API สำหรับผู้ให้บริการที่คุณต้องการใช้ ข้อมูลนี้จะถูกเก็บไว้ที่ ~/.local/share/closedcode/auth.json

closedcode auth login

เมื่อ vanilla-closedcode เริ่มต้น ระบบจะโหลดข้อมูลรับรองจากไฟล์นี้ ดังนั้นคุณจึงไม่ต้องกังวลกับการตั้งค่าตัวแปรสภาพแวดล้อมหรือไฟล์ .env ในโปรเจ็กต์ของคุณ

list

แสดงรายการข้อมูลรับรองการตรวจสอบสิทธิ์ทั้งหมดที่กำหนดค่าไว้

closedcode auth list

หรือใช้นามแฝงสั้น

closedcode auth ls

logout

ออกจากระบบของผู้ให้บริการและลบข้อมูลรับรองออกจากไฟล์ auth

closedcode auth logout

github

จัดการเอเจนต์ GitHub สำหรับเวิร์กโฟลว์อัตโนมัติ

closedcode github [command]

install

ติดตั้งเอเจนต์ GitHub ใน repository ของคุณ

closedcode github install

คำสั่งนี้จะสร้างเวิร์กโฟลว์ GitHub Actions และแนะนำคุณตลอดกระบวนการตั้งค่า เรียนรู้เพิ่มเติม

run

รันเอเจนต์ GitHub ด้วยตนเองหรือใน GitHub Actions

closedcode github run
แฟล็ก
แฟล็ก คำอธิบาย
{"--event"} เหตุการณ์ GitHub เพื่อทริกเกอร์เอเจนต์
{"--token"} GitHub token

mcp

จัดการเซิร์ฟเวอร์ Model Context Protocol (MCP)

closedcode mcp [command]

add

เพิ่มเซิร์ฟเวอร์ MCP ลงในการกำหนดค่าของคุณ

closedcode mcp add

คำสั่งนี้จะแนะนำคุณผ่านวิซาร์ดเพื่อกำหนดค่าเซิร์ฟเวอร์ MCP ใหม่

list

แสดงรายชื่อเซิร์ฟเวอร์ MCP ทั้งหมดที่กำหนดค่าไว้และสถานะ

closedcode mcp list

หรือใช้นามแฝงสั้น

closedcode mcp ls

auth

ยืนยันตัวตนกับเซิร์ฟเวอร์ MCP ที่ต้องการ OAuth

closedcode mcp auth [name]

หากไม่ระบุชื่อเซิร์ฟเวอร์ คุณจะได้รับแจ้งให้เลือกจากเซิร์ฟเวอร์ที่รองรับ OAuth ที่พร้อมใช้งาน

คุณยังสามารถแสดงรายการสถานะการตรวจสอบสิทธิ์ OAuth ได้:

closedcode mcp auth list

หรือใช้นามแฝงสั้น

closedcode mcp auth ls

logout

ลบข้อมูลรับรอง OAuth สำหรับเซิร์ฟเวอร์ MCP

closedcode mcp logout [name]

debug

ดีบักการเชื่อมต่อเซิร์ฟเวอร์ MCP

closedcode mcp debug <name>

models

แสดงรายการโมเดลที่พร้อมใช้งานทั้งหมดจากผู้ให้บริการที่กำหนดค่าไว้

closedcode models [provider]

คำสั่งนี้จะแสดง ID โมเดลทั้งหมดในรูปแบบ provider/model

คุณสามารถใช้ชื่อโมเดลเหล่านี้ในการกำหนดค่า config ของคุณ

คุณยังสามารถส่งผ่านรหัสผู้ให้บริการเพื่อกรองโมเดลตามผู้ให้บริการรายนั้นได้

closedcode models anthropic

แฟล็ก

แฟล็ก คำอธิบาย
{"--refresh"} รีเฟรชแคชโมเดลจาก models.dev
{"--verbose"} แสดงรายละเอียดโมเดลเพิ่มเติม (รวมข้อมูลเมตาเช่นต้นทุน)
{"--json"} แสดงผลลัพธ์เป็น JSON

ใช้แฟล็ก --refresh เพื่ออัปเดตรายการโมเดลที่แคชไว้ มีประโยชน์เมื่อมีการเพิ่มโมเดลใหม่ให้กับผู้ให้บริการและคุณต้องการเห็นใน vanilla-closedcode

closedcode models --refresh

run

รัน vanilla-closedcode โดยไม่ต้องใช้ TUI โดยส่งพรอมต์โดยตรง

closedcode run [message..]

มีประโยชน์สำหรับการทำงานอัตโนมัติหรือเมื่อต้องการคำตอบอย่างรวดเร็วโดยไม่ต้องเปิด TUI ตัวอย่าง:

closedcode run Explain the use of context in Go

คุณยังสามารถแนบไปกับเซิร์ฟเวอร์ closedcode serve ที่ทำงานอยู่ หรือใช้เซิร์ฟเวอร์ MCP แบบ headless:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

แฟล็ก

แฟล็ก สั้น คำอธิบาย
{"--command"} คำสั่งที่จะรัน (ใช้ส่วนที่เหลือของ args เป็นอาร์กิวเมนต์)
{"--continue"} -c ดำเนินการต่อจากเซสชันล่าสุด
{"--session"} -s ID เซสชันเพื่อดำเนินการต่อ
{"--fork"} แยกเซสชันเมื่อทำต่อ (ใช้กับ --continue หรือ --session)
{"--share"} สร้างลิงก์แชร์สำหรับเซสชัน
{"--model"} -m โมเดลที่จะใช้ในรูปแบบ provider/model
{"--agent"} เอเจนต์ที่จะใช้
{"--file"} -f แนบไฟล์ไปกับข้อความ
{"--format"} รูปแบบเอาต์พุต: text (จัดรูปแบบ) หรือ json (JSON ดิบ)
{"--title"} ชื่อสำหรับเซสชัน (หากไม่ได้ระบุ จะสร้างจากพรอมต์)
{"--attach"} แนบไปกับเซิร์ฟเวอร์ closedcode ที่ทำงานอยู่ (เช่น http://localhost:4096)
{"--password"} -p รหัสผ่านการยืนยันตัวตนพื้นฐาน (ค่าเริ่มต้นคือ CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u ชื่อผู้ใช้การยืนยันตัวตนพื้นฐาน (ค่าเริ่มต้นคือ CLOSEDCODE_SERVER_USERNAME หรือ closedcode)
{"--dir"} ไดเร็กทอรีสำหรับรัน หรือเส้นทางบนเซิร์ฟเวอร์ระยะไกลเมื่อแนบ
{"--variant"} ตัวแปรโมเดล (ระดับการใช้เหตุผลเฉพาะผู้ให้บริการ)
{"--thinking"} แสดงบล็อกความคิด
{"--port"} พอร์ตสำหรับเซิร์ฟเวอร์ภายในเครื่อง (หากไม่ได้ระบุ จะใช้พอร์ตสุ่ม)

serve

เริ่มต้นเซิร์ฟเวอร์ vanilla-closedcode headless ที่ให้บริการ API HTTP ดู เอกสารเซิร์ฟเวอร์ สำหรับรายละเอียด API เต็มรูปแบบ

closedcode serve

คำสั่งนี้จะเริ่มต้นเซิร์ฟเวอร์ HTTP ให้ API เข้าถึงฟังก์ชันการทำงานของ vanilla-closedcode ได้โดยไม่ต้องมี TUI นอกจากนี้ยังรองรับการตรวจสอบสิทธิ์พื้นฐาน HTTP (ชื่อผู้ใช้เริ่มต้นคือ closedcode และรหัสผ่านระบุโดย CLOSEDCODE_SERVER_PASSWORD)

แฟล็ก

แฟล็ก คำอธิบาย
{"--port"} พอร์ตที่จะฟัง
{"--hostname"} ชื่อโฮสต์ที่จะฟัง
{"--mdns"} เปิดใช้งานการค้นหา mDNS
{"--cors"} ต้นกำเนิดเพิ่มเติมที่อนุญาตสำหรับ CORS

ตัวแปรสภาพแวดล้อม

vanilla-closedcode สามารถกำหนดค่าโดยใช้ตัวแปรสภาพแวดล้อม

ตัวแปร ชนิด คำอธิบาย
CLOSEDCODE_AUTO_SHARE Boolean แชร์เซสชันอัตโนมัติเมื่อสร้าง
CLOSEDCODE_GIT_BASH_PATH String เส้นทางไปยัง Git Bash บน Windows
CLOSEDCODE_CONFIG String เส้นทางไปยังไฟล์การกำหนดค่า
CLOSEDCODE_TUI_CONFIG String เส้นทางไปยังไฟล์การกำหนดค่า TUI
CLOSEDCODE_CONFIG_DIR String เส้นทางไปยังไดเร็กทอรีการกำหนดค่า
CLOSEDCODE_CONFIG_CONTENT String เนื้อหาการกำหนดค่าแบบ inline JSON
CLOSEDCODE_DISABLE_AUTOUPDATE Boolean ปิดใช้งานการอัปเดตอัตโนมัติ
CLOSEDCODE_DISABLE_PRUNE Boolean ปิดใช้งานการลบข้อมูลเซสชันเก่า
CLOSEDCODE_DISABLE_TERMINAL_TITLE Boolean ปิดใช้งานการตั้งชื่อหน้าต่าง terminal
CLOSEDCODE_PERMISSION String การกำหนดค่าสิทธิ์แบบ inline JSON
