Windows (WSL)

ใช้ vanilla-closedcode บน Windows ผ่าน WSL ให้ลื่นที่สุด

แม้ว่า vanilla-closedcode จะรันบน Windows ได้โดยตรง แต่เราแนะนำให้ใช้ Windows Subsystem for Linux (WSL) เพื่อประสบการณ์ที่ดีที่สุด WSL ให้สภาพแวดล้อม Linux ที่ทำงานร่วมกับฟีเจอร์ของ vanilla-closedcode ได้อย่างราบรื่น

ทำไมต้องใช้ WSL?

WSL ให้ประสิทธิภาพระบบไฟล์ที่ดีกว่า รองรับ terminal ได้เต็มรูปแบบ และเข้ากันได้กับเครื่องมือพัฒนาที่ vanilla-closedcode ใช้งาน
---

การตั้งค่า

  1. ติดตั้ง WSL

    ถ้ายังไม่ได้ติดตั้ง ให้ทำตามคู่มือทางการของ Microsoft เพื่อ ติดตั้ง WSL

  2. ติดตั้ง vanilla-closedcode ใน WSL

    เมื่อตั้งค่า WSL เสร็จแล้ว ให้เปิด terminal WSL และติดตั้ง vanilla-closedcode ด้วย วิธีติดตั้ง แบบใดแบบหนึ่ง

    ดาวน์โหลด vanilla-closedcode จากหน้า Releases: github.com/informanellica/vanilla-closedcode/releases

  3. ใช้งาน vanilla-closedcode จาก WSL

    ไปที่ไดเรกทอรีโปรเจกต์ของคุณ (เข้าถึงไฟล์ Windows ผ่าน /mnt/c/, /mnt/d/ เป็นต้น) แล้วรัน vanilla-closedcode

    cd /mnt/c/Users/YourName/project
closedcode

เดสก์ท็อปแอป + เซิร์ฟเวอร์บน WSL

ถ้าคุณต้องการใช้ vanilla-closedcode Desktop แต่ให้เซิร์ฟเวอร์ทำงานบน WSL:

  1. เริ่มเซิร์ฟเวอร์ใน WSL โดยใช้ --hostname 0.0.0.0 เพื่อให้เชื่อมต่อจากภายนอกได้:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. เชื่อมต่อเดสก์ท็อปแอป ไปที่ http://localhost:4096

ถ้า `localhost` ใช้งานไม่ได้ในสภาพแวดล้อมของคุณ ให้ใช้ IP ของ WSL แทน (จาก WSL: `hostname -I`) แล้วเชื่อมต่อที่ `http://:4096`
เมื่อใช้ `--hostname 0.0.0.0` ให้ตั้งค่า `CLOSEDCODE_SERVER_PASSWORD` เพื่อป้องกันเซิร์ฟเวอร์ 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

เว็บไคลเอนต์ + WSL

เพื่อประสบการณ์เว็บที่ดีที่สุดบน Windows:

  1. รัน closedcode web ใน terminal WSL แทน PowerShell:

    closedcode web --hostname 0.0.0.0

  2. เข้าจากเบราว์เซอร์บน Windows ที่ http://localhost:<port> (vanilla-closedcode จะแสดง URL ให้)

การรัน closedcode web จาก WSL ช่วยให้เข้าถึงระบบไฟล์และใช้งาน terminal ได้ถูกต้อง พร้อมเปิดใช้งานผ่านเบราว์เซอร์ Windows ได้เหมือนเดิม

การเข้าถึงไฟล์ Windows

WSL สามารถเข้าถึงไฟล์ Windows ทั้งหมดของคุณได้ผ่านไดเรกทอรี /mnt/:

  • C: drive → /mnt/c/
  • D: drive → /mnt/d/
  • ไดรฟ์อื่นก็ใช้รูปแบบเดียวกัน

ตัวอย่าง:

cd /mnt/c/Users/YourName/Documents/project
closedcode
เพื่อให้ใช้งานได้ลื่นที่สุด แนะนำให้โคลนหรือคัดลอกรีโปไปไว้ในไฟล์ซิสเต็มของ WSL (เช่น `~/code/`) แล้วรัน vanilla-closedcode จากที่นั่น
---

เคล็ดลับ

  • แม้โปรเจกต์จะอยู่บนไดรฟ์ Windows ก็ควรรัน vanilla-closedcode ใน WSL เพื่อการเข้าถึงไฟล์ที่ลื่นกว่า
  • ใช้ vanilla-closedcode ควบคู่กับ WSL extension ของ VS Code เพื่อเวิร์กโฟลว์ที่ต่อเนื่อง
  • การตั้งค่าและเซสชันของ vanilla-closedcode จะถูกเก็บในสภาพแวดล้อม WSL ที่ ~/.local/share/closedcode/
