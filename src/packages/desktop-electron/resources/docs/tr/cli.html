CLI

closedcode CLI seçenekleri ve komutları.

closedcode CLI herhangi bir argüman olmadan çalıştırıldığında varsayılan olarak TUI'yi başlatır.

closedcode

Ancak bu sayfada belgelendiği gibi komutları da kabul eder. Bu, closedcode ile programlı olarak etkileşim kurmanıza olanak tanır.

closedcode run "Explain how closures work in JavaScript"

tui

closedcode terminal kullanıcı arayüzünü başlatır.

closedcode [project]

Bayraklar

Bayrak Kısa Açıklama
{"--continue"} -c Son oturuma devam et
{"--session"} -s Devam edecek oturum kimliği
{"--fork"} Devam ederken oturumu fork'lar (--continue veya --session ile kullanın)
{"--prompt"} Kullanılacak prompt
{"--model"} -m provider/model biçiminde kullanılacak model
{"--agent"} Kullanılacak agent
{"--port"} Dinlenecek port
{"--hostname"} Dinlenecek host adı

Komutlar

closedcode CLI ayrıca aşağıdaki komutlara da sahiptir.

agent

closedcode için agent'ları yönetin.

closedcode agent [command]

attach

serve veya web komutlarıyla başlatılan, halihazırda çalışan bir closedcode arka uç sunucusuna bir terminal ekleyin.

closedcode attach [url]

Bu, TUI öğesinin uzak bir closedcode arka ucuyla kullanılmasına olanak tanır. Örneğin:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Bayraklar

Bayrak Kısa Tanım
{"--dir"} TUI'yi başlatmak için çalışma dizini
{"--continue"} -c Son oturuma devam et
{"--session"} -s Devam edecek oturum açma bilgileri
{"--fork"} Devam ederken oturumu çatalla (--continue veya --session ile kullanın)
{"--password"} -p Temel kimlik doğrulama parolası (varsayılan: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Temel kimlik doğrulama kullanıcı adı (varsayılan: CLOSEDCODE_SERVER_USERNAME veya closedcode)

create

Özel yapılandırmaya sahip yeni bir agent oluşturun.

closedcode agent create

Bu komut, özel sistem istemi ve araç yapılandırmasıyla yeni bir agent oluşturma konusunda size yol gösterecektir.

list

Mevcut tüm agent'ları listeleyin.

closedcode agent list

auth

Sağlayıcılar için kimlik bilgilerini ve oturum açmayı yönetme komutu.

closedcode auth [command]

login

closedcode, Models.dev adresindeki sağlayıcı listesi tarafından desteklenmektedir, dolayısıyla kullanmak istediğiniz herhangi bir sağlayıcı için API anahtarlarını ayarlamak üzere closedcode auth login kullanabilirsiniz. Bu ~/.local/share/closedcode/auth.json'de saklanıyor.

closedcode auth login

closedcode başlatıldığında sağlayıcıları kimlik bilgileri dosyasından yükler. Ayrıca ortam değişkenlerindeki veya projenizdeki .env dosyasındaki anahtarları da kullanır.

list

Kimlik bilgileri dosyasında depolanan tüm kimliği doğrulanmış sağlayıcıları listeler.

closedcode auth list

Veya kısa versiyonu.

closedcode auth ls

logout

Bir sağlayıcıyı kimlik bilgileri dosyasından temizleyerek oturumunuzu kapatır.

closedcode auth logout

github

Depo otomasyonu için GitHub aracısını yönetin.

closedcode github [command]

install

GitHub aracısını deponuza yükleyin.

closedcode github install

Bu komut gerekli GitHub Actions workflow'unu kurar ve yapılandırma adımlarında size rehberlik eder. Daha fazla bilgi.

run

GitHub aracısını çalıştırın. Bu genellikle GitHub Eylemlerinde kullanılır.

closedcode github run
Bayraklar
Bayrak Açıklama
{"--event"} Aracıyı çalıştırmak için GitHub sahte olayı
{"--token"} GitHub personal access token

mcp

Model Bağlam Protokolü sunucularını yönetin.

closedcode mcp [command]

add

Yapılandırmanıza bir MCP sunucusu ekleyin.

closedcode mcp add

Bu komut, yerel veya uzak bir MCP sunucusu ekleme konusunda size yol gösterecektir.

list

Yapılandırılmış tüm MCP sunucularını ve bağlantı durumlarını listeleyin.

closedcode mcp list

Veya kısa versiyonunu kullanın.

closedcode mcp ls

auth

OAuth'un etkin olduğu bir MCP sunucusuyla kimlik doğrulaması yapın.

closedcode mcp auth [name]

Sunucu adı belirtmezseniz mevcut OAuth özellikli sunucular arasından seçim yapmanız istenir.

Ayrıca OAuth özellikli sunucuları ve bunların kimlik doğrulama durumlarını da listeleyebilirsiniz.

closedcode mcp auth list

Veya kısa versiyonunu kullanın.

closedcode mcp auth ls

logout

MCP sunucusu için OAuth kimlik bilgilerini kaldırın.

closedcode mcp logout [name]

debug

MCP sunucusu için OAuth bağlantı sorunlarının hatalarını ayıklayın.

closedcode mcp debug <name>

models

Yapılandırılmış sağlayıcıların tüm mevcut modellerini listeleyin.

closedcode models [provider]

Bu komut, yapılandırdığınız sağlayıcılardaki modelleri provider/model formatında listeler.

Bu, config dosyanızda kullanmanız gereken tam model adını bulmak için kullanışlıdır.

Modelleri bu sağlayıcıya göre filtrelemek için isteğe bağlı olarak bir sağlayıcı kimliğini iletebilirsiniz.

closedcode models anthropic

Bayraklar

Bayrak Tanım
{"--refresh"} Modeller.dev'den model önbelleğini yenileyin
{"--verbose"} Daha ayrıntılı model çıktısı kullanın (maliyetler gibi meta veriler içerir)

Önbelleğe alınan model listesini güncellemek için --refresh bayrağını kullanın. Bu, bir sağlayıcıya yeni modeller eklendiğinde ve bunları closedcode'da görmek istediğinizde kullanışlıdır.

closedcode models --refresh

run

Doğrudan bir istem ileterek closedcode'u etkileşimli olmayan modda çalıştırın.

closedcode run [message..]

Bu, komut dosyası oluşturma, otomasyon veya TUI'un tamamını başlatmadan hızlı bir yanıt istediğinizde kullanışlıdır. Örneğin.

closedcode run Explain the use of context in Go

Ayrıca, her çalıştırmada MCP sunucusunun soğuk önyükleme sürelerini önlemek için çalışan bir closedcode serve örneğine de ekleyebilirsiniz:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Bayraklar

Bayrak Kısa Açıklama
{"--command"} Çalıştırılacak komut, args için mesajı kullanın
{"--continue"} -c Son oturuma devam et
{"--session"} -s Devam edecek oturum kimliği
{"--fork"} Devam ederken oturumu fork'lar (--continue veya --session ile kullanın)
{"--share"} Oturumu paylaşın
{"--model"} -m provider/model biçiminde kullanılacak model
{"--agent"} Kullanılacak temsilci
{"--file"} -f Mesaja eklenecek dosya(lar)
{"--format"} Biçim: varsayılan (biçimlendirilmiş) veya json (ham JSON olayları)
{"--title"} Oturumun başlığı (değer sağlanmazsa kısaltılmış bilgi istemi kullanılır)
{"--attach"} Çalışan bir closedcode sunucusuna ekleyin (ör. http://localhost:4096)
{"--password"} -p Temel kimlik doğrulama parolası (varsayılan: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Temel kimlik doğrulama kullanıcı adı (varsayılan: CLOSEDCODE_SERVER_USERNAME veya closedcode)
{"--dir"} Çalıştırılacak dizin veya bağlanırken uzak sunucudaki yol
{"--variant"} Model varyantı (sağlayıcıya özgü muhakeme düzeyi)
{"--thinking"} Düşünme bloklarını göster
{"--port"} Yerel sunucunun bağlantı noktası (varsayılan olarak rastgele bağlantı noktasıdır)

serve

API erişimi için headless bir closedcode sunucusu başlatır. Tam HTTP arayüzü için server docs sayfasına bakın.

closedcode serve

Bu, TUI arayüzü olmadan closedcode işlevselliğine API erişimi sağlayan bir HTTP sunucusunu başlatır. HTTP temel kimlik doğrulamasını etkinleştirmek için CLOSEDCODE_SERVER_PASSWORD öğesini ayarlayın (kullanıcı adı varsayılan olarak closedcode şeklindedir).

Bayraklar

Bayrak Tanım
{"--port"} Dinlenecek bağlantı noktası
{"--hostname"} Dinlenecek ana bilgisayar adı
{"--mdns"} mDNS bulmayı etkinleştir
{"--cors"} CORS'a izin verecek ek tarayıcı kaynakları

session

closedcode oturumlarını yönetin.

closedcode session [command]

list

Tüm closedcode oturumlarını listeleyin.

closedcode session list
Bayraklar
Bayrak Kısa Tanım
{"--max-count"} -n En son N oturumla sınırla
{"--format"} Çıkış formatı: tablo veya json (tablo)

stats

closedcode oturumlarınız için belirteç kullanımı ve maliyet istatistiklerini gösterin.

closedcode stats

Bayraklar

Bayrak Açıklama
{"--days"} Son N güne ait istatistikleri göster (tüm zamanlar)
{"--tools"} Gösterilecek araç sayısı (tümü)
{"--models"} Model kullanım dökümünü göster (varsayılan olarak gizlidir). En üstteki N'yi göstermek için bir sayı iletin
{"--project"} Projeye göre filtrele (tüm projeler, boş değer: mevcut proje)

export

Oturum verilerini JSON olarak dışa aktarın.

closedcode export [sessionID]

Bir oturum kimliği sağlamazsanız mevcut oturumlar arasından seçim yapmanız istenir.

import

Bir JSON dosyasından veya closedcode paylaşımından URL oturum verilerini içe aktarın.

closedcode import <file>

Yerel bir dosyadan veya closedcode paylaşımından (URL) içe aktarabilirsiniz.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

Web arayüzüyle başsız bir closedcode sunucusu başlatın.

closedcode web

Bu, bir HTTP sunucusunu başlatır ve bir web arayüzü aracılığıyla closedcode'a erişmek için bir web tarayıcısı açar. HTTP temel kimlik doğrulamasını etkinleştirmek için CLOSEDCODE_SERVER_PASSWORD öğesini ayarlayın (kullanıcı adı varsayılan olarak closedcode şeklindedir).

Bayraklar

Bayrak Tanım
{"--port"} Dinlenecek bağlantı noktası
{"--hostname"} Dinlenecek ana bilgisayar adı
{"--mdns"} mDNS bulmayı etkinleştir
{"--cors"} CORS'a izin verecek ek tarayıcı kaynakları

acp

Bir ACP (Ajan İstemci Protokolü) sunucusu başlatın.

closedcode acp

Bu komut, nd-JSON kullanarak stdin/stdout aracılığıyla iletişim kuran bir ACP sunucusunu başlatır.

Bayraklar

Bayrak Açıklama
{"--cwd"} Çalışma dizini
{"--port"} Dinlenecek port
{"--hostname"} Dinlenecek host adı

uninstall

closedcode'u kaldırın ve ilgili tüm dosyaları kaldırın.

closedcode uninstall

Bayraklar

Bayrak Kısa Tanım
{"--keep-config"} -c Yapılandırma dosyalarını sakla
{"--keep-data"} -d Oturum verilerini ve anlık görüntüleri saklayın
{"--dry-run"} Nelerin kaldırılmadan kaldırılacağı göster
{"--force"} -f Onay istemlerini atla

upgrade

closedcode'u en son sürüme veya belirli bir sürüme günceller.

closedcode upgrade [target]

En son sürüme yükseltmek için.

closedcode upgrade

Belirli bir sürüme yükseltmek için:

closedcode upgrade v0.1.48

Bayrak Kısa Açıklama
{"--method"} -m Kullanılan kurulum yöntemi: curl, npm, pnpm, brew
