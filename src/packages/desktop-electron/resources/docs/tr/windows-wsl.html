Windows (WSL)

En iyi deneyim icin closedcode'u WSL'de calistirin.

closedcode Windows'ta dogrudan calisabilir, ancak en iyi deneyim icin Windows Subsystem for Linux (WSL) kullanmanizi oneriyoruz. WSL, closedcode ozellikleriyle sorunsuz calisan bir Linux ortami saglar.

Neden WSL?

WSL, daha iyi dosya sistemi performansi, tam terminal destegi ve closedcode'un dayandigi gelistirme araclariyla uyumluluk sunar.
---

Kurulum

  1. WSL'i yukleyin

    Henuz yuklemediyseniz resmi Microsoft rehberini kullanarak WSL kurulumunu yapin.

  2. closedcode'u WSL icine yukleyin

    WSL hazir olduktan sonra WSL terminalini acin ve closedcode'u kurulum yontemlerinden biriyle yukleyin.

    curl -fsSL https://github.com/informanellica/vanilla-closedcode/releases | bash

  3. closedcode'u WSL icinden kullanin

    Proje dizininize gidin (/mnt/c/, /mnt/d/ vb. ile Windows dosyalarina erisebilirsiniz) ve closedcode'u calistirin.

    cd /mnt/c/Users/YourName/project
closedcode

Masaüstü uygulaması + WSL sunucusu

closedcode Desktop kullanmak isteyip sunucuyu WSL'de calistirmak istiyorsaniz:

  1. Sunucuyu WSL'de baslatin ve dis baglantilara izin vermek icin --hostname 0.0.0.0 kullanin:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Desktop uygulamasini http://localhost:4096 adresine baglayin

Kurulumunuzda `localhost` calismiyorsa bunun yerine WSL IP adresini kullanin (WSL icinde: `hostname -I`) ve `http://:4096` adresine baglanin.
`--hostname 0.0.0.0` kullaniyorsaniz sunucuyu guvene almak icin `CLOSEDCODE_SERVER_PASSWORD` ayarlayin. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Web istemcisi + WSL

Windows'ta en iyi web deneyimi icin:

  1. closedcode web komutunu PowerShell yerine WSL terminalinde calistirin:

    closedcode web --hostname 0.0.0.0

  2. Windows tarayicinizdan http://localhost:<port> adresini acin (closedcode URL'yi yazdirir)

closedcode web komutunu WSL'den calistirmak, dosya sistemi erisimi ve terminal entegrasyonunu daha tutarli hale getirirken tarayicidan erisimi korur.

Windows dosyalarına erişin

WSL, tum Windows dosyalariniza /mnt/ dizini uzerinden erisebilir:

  • C: surucusu -> /mnt/c/
  • D: surucusu -> /mnt/d/
  • Ve digerleri...

Ornek:

cd /mnt/c/Users/YourName/Documents/project
closedcode
En sorunsuz deneyim icin deponuzu WSL dosya sistemine (ornegin `~/code/` altina) klonlayip closedcode'u orada calistirmayi dusunun.
---

İpuçları

  • Windows suruculerinde duran projelerde closedcode'u WSL icinde calistirin - dosya erisimi daha sorunsuz olur
  • Butunlesik bir gelistirme akisi icin closedcode ile birlikte VS Code WSL eklentisini kullanin
  • closedcode konfigurasyonunuz ve oturumlariniz WSL ortaminda ~/.local/share/closedcode/ konumunda saklanir
