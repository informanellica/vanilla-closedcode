Windows (WSL)

WSL로 Windows에서 closedcode를 더 편하게 사용하세요.

closedcode는 Windows에서 직접 실행할 수 있지만, 가장 좋은 사용 경험을 위해 Windows Subsystem for Linux (WSL) 사용을 권장합니다. WSL은 closedcode 기능과 자연스럽게 동작하는 Linux 환경을 제공합니다.

왜 WSL인가요?

WSL은 더 나은 파일 시스템 성능, 완전한 terminal 지원, 그리고 closedcode가 의존하는 개발 도구와의 호환성을 제공합니다.
---

설정

  1. WSL 설치하기

    아직 설치하지 않았다면 Microsoft 공식 가이드를 따라 WSL을 설치하세요.

  2. WSL에 closedcode 설치하기

    WSL 설정이 끝나면 WSL terminal을 열고 설치 방법 중 하나로 closedcode를 설치하세요.

    Releases 페이지에서 빌드를 내려받아 설치하세요: github.com/informanellica/vanilla-closedcode/releases

  3. WSL에서 closedcode 실행하기

    프로젝트 디렉터리로 이동한 뒤(closedcode에서 Windows 파일은 /mnt/c/, /mnt/d/ 등으로 접근) closedcode를 실행하세요.

    cd /mnt/c/Users/YourName/project
closedcode

데스크톱 앱 + WSL 서버

closedcode 데스크톱 앱을 쓰면서 서버는 WSL에서 실행하고 싶다면 다음처럼 진행하세요.

  1. WSL에서 서버 시작하기

    외부 연결을 허용하려면 --hostname 0.0.0.0 옵션을 사용하세요.

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. 데스크톱 앱 연결하기

    http://localhost:4096로 연결하세요.

환경에 따라 `localhost`가 동작하지 않으면 WSL의 IP 주소를 사용하세요(WSL에서 `hostname -I` 실행). 이후 `http://:4096`로 접속하면 됩니다.
`--hostname 0.0.0.0`을 사용할 때는 `CLOSEDCODE_SERVER_PASSWORD`를 설정해 서버를 보호하세요. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

웹 클라이언트 + WSL

Windows에서 웹 사용 경험을 가장 좋게 하려면:

  1. PowerShell 대신 WSL terminal에서 closedcode web 실행하기

    closedcode web --hostname 0.0.0.0

  2. Windows 브라우저에서 접속하기

    http://localhost:<port>로 접속하세요(closedcode가 URL을 출력합니다).

WSL에서 closedcode web를 실행하면 올바른 파일 시스템 접근과 terminal 통합을 유지하면서도 Windows 브라우저에서 바로 사용할 수 있습니다.

Windows 파일 접근

WSL에서는 /mnt/ 디렉터리를 통해 Windows 파일 전체에 접근할 수 있습니다.

  • C: drive → /mnt/c/
  • D: drive → /mnt/d/
  • 다른 드라이브도 같은 방식입니다

예시:

cd /mnt/c/Users/YourName/Documents/project
closedcode
가장 매끄러운 경험을 원한다면 저장소를 WSL 파일시스템(예: `~/code/`)으로 복제하거나 복사한 뒤 그 위치에서 closedcode를 실행해 보세요.
---

팁

  • Windows 드라이브에 있는 프로젝트라도 closedcode는 WSL에서 실행하면 파일 접근이 더 자연스럽습니다
  • 통합된 개발 워크플로를 위해 closedcode와 함께 VS Code의 WSL 확장을 사용해 보세요
  • closedcode 설정과 세션은 WSL 환경의 ~/.local/share/closedcode/에 저장됩니다
