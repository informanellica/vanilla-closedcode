CLI

vanilla-closedcode CLI opcije i naredbe.

vanilla-closedcode CLI po defaultu pokreće TUI kada se pokrene bez ikakvih argumenata.

closedcode

Ali takođe prihvata naredbe kao što je dokumentovano na ovoj stranici. Ovo vam omogućava programsku interakciju sa vanilla-closedcode.

closedcode run "Explain how closures work in JavaScript"

tui

Pokrenite vanilla-closedcode terminalski korisnički interfejs.

closedcode [project]

Opcije

Opcija Kratko Opis
{"--continue"} -c Nastavite posljednju sesiju
{"--session"} -s ID sesije za nastavak
{"--fork"} Forkujte sesiju pri nastavku (koristiti sa --continue ili --session)
{"--prompt"} Prompt za upotrebu
{"--model"} -m Model za korištenje u obliku provider/model
{"--agent"} Agent za korištenje
{"--port"} Port na kojem treba slušati
{"--hostname"} Hostname na kojem treba slušati

Naredbe

vanilla-closedcode CLI takođe ima sljedeće naredbe.

agent

Upravljajte vanilla-closedcode agentima.

closedcode agent [command]

attach

Priključite terminal na već pokrenut vanilla-closedcode backend server pokrenut putem serve ili web naredbi.

closedcode attach [url]

Ovo omogućava korištenje TUI-ja sa udaljenim vanilla-closedcode backend-om. Na primjer:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Opcije

Opcija Kratko Opis
{"--dir"} Radni direktorij za pokretanje TUI-a
{"--continue"} -c Nastavi posljednju sesiju
{"--session"} -s ID sesije za nastavak
{"--fork"} Forkuj sesiju prilikom nastavka (koristite sa --continue ili --session)
{"--password"} -p Lozinka za osnovnu autentifikaciju (zadano: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Korisničko ime za osnovnu autentifikaciju (zadano: CLOSEDCODE_SERVER_USERNAME ili closedcode)

create

Kreirajte novog agenta s prilagođenom konfiguracijom.

closedcode agent create

Ova naredba će vas voditi kroz kreiranje novog agenta sa prilagođenim sistemskim promptom i konfiguracijom alata.

list

Navedite sve dostupne agente.

closedcode agent list

auth

Naredba za upravljanje vjerodajnicama i prijavom za provajdere.

closedcode auth [command]

login

vanilla-closedcode pokreće lista provajdera na Models.dev, tako da možete koristiti closedcode auth login da konfigurirate API ključeve za bilo kojeg provajdera kojeg želite koristiti. Ovo je pohranjeno u ~/.local/share/closedcode/auth.json.

closedcode auth login

Kada se vanilla-closedcode pokrene, učitava dobavljače iz datoteke vjerodajnica. I ako postoje neki ključevi definirani u vašim okruženjima ili .env fajl u vašem projektu.

list

Navodi sve autentifikovane dobavljače pohranjene u datoteci vjerodajnica.

closedcode auth list

Ili kratka verzija.

closedcode auth ls

logout

Odjavljuje vas s provajdera tako što ga briše iz datoteke vjerodajnica.

closedcode auth logout

github

Upravljajte GitHub agentom za automatizaciju repozitorija.

closedcode github [command]

install

Instalirajte GitHub agenta u svoj repozitorij.

closedcode github install

Ovo postavlja neophodni tok rada GitHub Actions i vodi vas kroz proces konfiguracije. Saznajte više.

run

Pokrenite GitHub agent. Ovo se obično koristi u GitHub Actions.

closedcode github run
Opcije
Opcija Opis
{"--event"} GitHub mock event za pokretanje agenta
{"--token"} GitHub Personal Access Token

mcp

Upravljajte Model Context Protocol (MCP) serverima.

closedcode mcp [command]

add

Dodajte MCP server svojoj konfiguraciji.

closedcode mcp add

Ova naredba će vas voditi kroz dodavanje lokalnog ili udaljenog MCP servera.

list

Navedite sve konfigurirane MCP servere i njihov status veze.

closedcode mcp list

Ili koristite kratku verziju.

closedcode mcp ls

auth

Autentifikujte se sa MCP serverom koji je omogućen za OAuth.

closedcode mcp auth [name]

Ako ne navedete ime servera, od vas će biti zatraženo da izaberete neki od dostupnih servera koji podržavaju OAuth. Također možete navesti servere koji podržavaju OAuth i njihov status autentifikacije.

closedcode mcp auth list

Ili koristite kratku verziju.

closedcode mcp auth ls

logout

Uklonite OAuth vjerodajnice za MCP server.

closedcode mcp logout [name]

debug

Otklanjanje grešaka (debug) OAuth veze sa MCP serverom.

closedcode mcp debug <name>

models

Navedite sve dostupne modele konfiguriranih provajdera.

closedcode models [provider]

Ova naredba prikazuje sve modele dostupne kod vaših konfiguriranih provajdera u formatu provider/model. Ovo je korisno za pronalaženje tačnog naziva modela za korištenje u vašoj konfiguraciji. Opciono možete proslijediti ID provajdera za filtriranje modela po tom dobavljaču.

closedcode models anthropic

Opcije

Opcija Opis
{"--refresh"} Osvježite keš modela sa models.dev
{"--verbose"} Koristite detaljniji izlaz modela (uključuje metapodatke poput troškova)

Koristite --refresh zastavicu da ažurirate keširanu listu modela. Ovo je korisno kada su novi modeli dodani provajderu i želite da ih vidite u vanilla-closedcode.

closedcode models --refresh

run

Pokrenite vanilla-closedcode u neinteraktivnom modu tako što ćete direktno proslijediti prompt.

closedcode run [message..]

Ovo je korisno za skriptiranje, automatizaciju ili kada želite brz odgovor bez pokretanja punog TUI-ja. Na primjer:

closedcode run Explain the use of context in Go

Također možete priključiti pokrenutu closedcode serve instancu kako biste izbjegli vrijeme hladnog pokretanja MCP servera pri svakom pokretanju:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Opcije

Opcija Kratko Opis
{"--command"} Naredba za pokretanje, koristite poruku za argumente
{"--continue"} -c Nastavite posljednju sesiju
{"--session"} -s ID sesije za nastavak
{"--fork"} Forkujte sesiju pri nastavku (koristiti sa --continue ili --session)
{"--share"} Podijelite sesiju
{"--model"} -m Model za korištenje u obliku provider/model
{"--agent"} Agent za korištenje
{"--file"} -f Fajlovi koje treba priložiti poruci
{"--format"} Format: default (formatiran) ili json (sirovi JSON događaji)
{"--title"} Naslov sesije (koristi skraćeni prompt ako nije navedena vrijednost)
{"--attach"} Priključite na pokrenuti vanilla-closedcode server (npr. http://localhost:4096)
{"--password"} -p Lozinka za osnovnu autentifikaciju (zadano: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Korisničko ime za osnovnu autentifikaciju (zadano: CLOSEDCODE_SERVER_USERNAME ili closedcode)
{"--dir"} Direktorij za pokretanje, ili putanja na udaljenom serveru pri spajanju
{"--variant"} Varijanta modela (napor zaključivanja specifičan za provajdera)
{"--thinking"} Prikaži blokove razmišljanja
{"--port"} Port za lokalni server (zadano na nasumični port)

serve

Pokrenite vanilla-closedcode headless server za API pristup. Pogledajte server docs za kompletan HTTP interfejs.

closedcode serve

Ovo pokreće HTTP server koji pruža API pristup funkcionalnosti vanilla-closedcode-a bez TUI interfejsa. Postavite CLOSEDCODE_SERVER_PASSWORD da omogućite HTTP osnovnu auth (korisničko ime je zadano na closedcode).

Opcije

Opcija Opis
{"--port"} Port na kojem treba slušati
{"--hostname"} Hostname na kojem treba slušati
{"--mdns"} Omogući mDNS otkrivanje
{"--cors"} Dodatni origin(i) pretraživača koji dozvoljavaju CORS

closedcode session [command]

list

closedcode session list
Opcije

stats

Prikaži statistiku upotrebe tokena i troškova za vaše vanilla-closedcode sesije.

closedcode stats

Opcije

Opcija Opis
{"--days"} Prikaži statistiku za zadnjih N dana (sva vremena)
{"--tools"} Broj alata za prikaz (svi)
{"--models"} Prikaži raščlambu korištenja modela (skriveno prema zadanim postavkama). Proslijedite broj za prikaz top N
{"--project"} Filtriraj po projektu (svi projekti, prazan niz: trenutni projekt)

export

Izvezite podatke sesije kao JSON.

closedcode export [sessionID]

Ako ne unesete ID sesije, od vas će biti zatraženo da odaberete neku od dostupnih sesija.

import

Uvezite podatke sesije iz JSON datoteke ili vanilla-closedcode dijeljenog URL-a.

closedcode import <file>

Možete uvesti iz lokalne datoteke ili vanilla-closedcode dijeljenog URL-a.

closedcode import session.json
closedcode import https://share.example.com/s/abc123
