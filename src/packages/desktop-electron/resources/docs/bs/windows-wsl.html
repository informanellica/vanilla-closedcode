Windows (WSL)

Pokrenite vanilla-closedcode na Windowsu preko WSL-a.

Iako vanilla-closedcode moze raditi direktno na Windowsu, preporucujemo Windows Subsystem for Linux (WSL) za najbolje iskustvo. WSL daje Linux okruzenje koje glatko radi sa vanilla-closedcode funkcijama.

Zašto WSL?

WSL nudi bolje performanse fajl sistema, punu terminalsku podrsku i kompatibilnost s razvojnim alatima na koje se vanilla-closedcode oslanja.
---

Postavljanje

  1. Instalirajte WSL

    Ako vec niste, instalirajte WSL prema zvanicnom Microsoft vodicu.

  2. Instalirajte vanilla-closedcode u WSL-u

    Kad je WSL spreman, otvorite WSL terminal i preuzmite vanilla-closedcode sa stranice izdanja: github.com/informanellica/vanilla-closedcode/releases.

  3. Koristite vanilla-closedcode iz WSL-a

    Idite u direktorij projekta (Windows fajlovima pristupate preko /mnt/c/, /mnt/d/ itd.) i pokrenite vanilla-closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

Desktop aplikacija + WSL Server

Ako preferirate vanilla-closedcode Desktop aplikaciju, ali zelite da server radi u WSL-u:

  1. Pokrenite server u WSL-u sa --hostname 0.0.0.0 da dozvolite vanjske konekcije:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Povezite Desktop aplikaciju na http://localhost:4096

Ako `localhost` ne radi u vasem setupu, povezte se preko WSL IP adrese (u WSL-u: `hostname -I`) i koristite `http://:4096`.
Kada koristite `--hostname 0.0.0.0`, postavite `CLOSEDCODE_SERVER_PASSWORD` da zastitite server. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Web klijent + WSL

Za najbolje web iskustvo na Windowsu:

  1. Pokrenite closedcode web u WSL terminalu umjesto u PowerShell-u:

    closedcode web --hostname 0.0.0.0

  2. Otvorite iz Windows browsera na http://localhost:<port> (vanilla-closedcode ispisuje URL)

Pokretanje closedcode web iz WSL-a osigurava ispravan pristup fajl sistemu i terminalsku integraciju, a i dalje je dostupno iz Windows browsera.

Pristup Windows fajlovima

WSL moze pristupiti svim Windows fajlovima kroz /mnt/ direktorij:

  • Disk C: → /mnt/c/
  • Disk D: → /mnt/d/
  • I tako dalje...

Primjer:

cd /mnt/c/Users/YourName/Documents/project
closedcode
Za najgladje iskustvo, razmislite da klonirate/kopirate repo u WSL fajl sistem (npr. pod `~/code/`) i tu pokrenete vanilla-closedcode.
---

Savjeti

  • Drzite vanilla-closedcode u WSL-u za projekte na Windows diskovima - pristup fajlovima je jednostavan
  • Koristite VS Code WSL ekstenziju uz vanilla-closedcode za integrisan tok rada
  • Vase vanilla-closedcode konfiguracije i sesije cuvaju se u WSL okruzenju na ~/.local/share/closedcode/
