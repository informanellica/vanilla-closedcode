CLI

vanilla-closedcode CLI-alternativer og kommandoer.

vanilla-closedcode CLI starter som standard TUI når den kjøres uten noen argumenter.

closedcode

Men den godtar også kommandoer som dokumentert på denne siden. Dette lar deg samhandle med vanilla-closedcode programmatisk.

closedcode run "Explain how closures work in JavaScript"

tui

Start vanilla-closedcode-terminalbrukergrensesnittet.

closedcode [project]

Flagg

Flagg Kort Beskrivelse
{"--continue"} -c Fortsett siste økt
{"--session"} -s Økt ID for å fortsette
{"--fork"} Forgren økten ved fortsettelse (bruk med --continue eller --session)
{"--prompt"} Ledetekst som skal brukes
{"--model"} -m Modell å bruke i form av leverandør/modell
{"--agent"} Agent som skal brukes
{"--port"} Port å lytte på
{"--hostname"} Vertsnavn å lytte på

Kommandoer

vanilla-closedcode CLI har også følgende kommandoer.

agent

Administrer agenter for vanilla-closedcode.

closedcode agent [command]

attach

Koble en terminal til en allerede kjørende vanilla-closedcode-backend-server startet via serve eller web kommandoer.

closedcode attach [url]

Dette gjør det mulig å bruke TUI med en ekstern vanilla-closedcode-backend. For eksempel:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Flagg

Flagg Kort Beskrivelse
{"--dir"} Arbeidskatalog for å starte TUI i
{"--continue"} -c Fortsett siste økt
{"--session"} -s Økt ID for å fortsette
{"--fork"} Forgren økten ved fortsettelse (bruk med --continue eller --session)
{"--password"} -p Passord for grunnleggende autentisering (standard: CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Brukernavn for grunnleggende autentisering (standard: CLOSEDCODE_SERVER_USERNAME eller closedcode)

create

Opprett en ny agent med tilpasset konfigurasjon.

closedcode agent create

Denne kommandoen vil veilede deg gjennom å lage en ny agent med en tilpasset systemmelding og verktøykonfigurasjon.

list

List opp alle tilgjengelige agenter.

closedcode agent list

auth

Kommando for administrasjon av påloggingsinformasjon og innlogging for leverandører.

closedcode auth [command]

login

vanilla-closedcode drives av leverandørlisten på Models.dev, så du kan bruke closedcode auth login til å konfigurere API-nøkler for enhver leverandør du vil bruke. Dette er lagret i ~/.local/share/closedcode/auth.json.

closedcode auth login

Når vanilla-closedcode starter opp, laster den leverandørene fra filen med påloggingsinformasjon. Og hvis det er noen nøkler definert i miljøene dine eller en .env-fil i prosjektet ditt.

list

Viser alle de autentiserte leverandørene som er lagret i filen med påloggingsinformasjon.

closedcode auth list

Eller bruk kortversjonen.

closedcode auth ls

logout

Logger deg ut av en leverandør ved å fjerne den fra filen med påloggingsinformasjon.

closedcode auth logout

github

Administrer GitHub-agenten for depotautomatisering.

closedcode github [command]

install

Installer GitHub-agenten i depotet ditt.

closedcode github install

Dette setter opp den nødvendige GitHub Actions-arbeidsflyten og veileder deg gjennom konfigurasjonsprosessen. Finn ut mer.

run

Kjør GitHub-agenten. Dette brukes vanligvis i GitHub Actions.

closedcode github run
Flagg
Flagg Beskrivelse
{"--event"} GitHub mock-hendelse agenten skal kjøres for
{"--token"} GitHub personlig tilgangsnøkkel

mcp

Administrer Model Context Protocol-servere.

closedcode mcp [command]

add

Legg til en MCP-server til konfigurasjonen.

closedcode mcp add

Denne kommandoen vil veilede deg gjennom å legge til enten en lokal eller ekstern MCP-server.

list

Liste alle konfigurerte MCP-servere og deres tilkoblingsstatus.

closedcode mcp list

Eller bruk den korte versjonen.

closedcode mcp ls

auth

Autentiser med en OAuth-aktivert MCP-server.

closedcode mcp auth [name]

Hvis du ikke oppgir et servernavn, blir du bedt om å velge blant tilgjengelige OAuth-kompatible servere.

Du kan også liste opp OAuth-kompatible servere og deres autentiseringsstatus.

closedcode mcp auth list

Eller bruk den korte versjonen.

closedcode mcp auth ls

logout

Fjern OAuth-påloggingsinformasjon for en MCP-server.

closedcode mcp logout [name]

debug

Feilsøk OAuth-tilkoblingsproblemer for en MCP-server.

closedcode mcp debug <name>

models

List opp alle tilgjengelige modeller fra konfigurerte leverandører.

closedcode models [provider]

Denne kommandoen viser alle modeller som er tilgjengelige på tvers av dine konfigurerte leverandører i formatet provider/model.

Dette er nyttig for å finne ut det eksakte modellnavnet som skal brukes i din konfigurasjon.

Du kan eventuelt sende en leverandør ID for å filtrere modeller etter den leverandøren.

closedcode models anthropic

Flagg

Flagg Beskrivelse
{"--refresh"} Oppdater modellbufferen fra models.dev
{"--verbose"} Bruk mer detaljert modellutdata (inkluderer metadata som kostnader)

Bruk --refresh-flagget for å oppdatere den bufrede modelllisten. Dette er nyttig når nye modeller er lagt til en leverandør og du vil se dem i vanilla-closedcode.

closedcode models --refresh

run

Kjør vanilla-closedcode i ikke-interaktiv modus ved å sende en melding direkte.

closedcode run [message..]

Dette er nyttig for skripting, automatisering eller når du vil ha et raskt svar uten å starte hele TUI. For eksempel.

closedcode run Explain the use of context in Go

Du kan også koble til en kjørende closedcode serve-forekomst for å unngå MCP-serveres kaldstarttid ved hver kjøring:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

serve

Start en headless vanilla-closedcode-server for API-tilgang. Sjekk ut server-dokumentene for hele HTTP-grensesnittet.

closedcode serve

Dette starter en HTTP-server som gir API tilgang til vanilla-closedcode-funksjonalitet uten TUI-grensesnittet. Sett CLOSEDCODE_SERVER_PASSWORD for å aktivere HTTP grunnleggende autentisering (brukernavn er standard til closedcode).

Flagg

Flagg Beskrivelse
{"--port"} Port å lytte på
{"--hostname"} Vertsnavn å lytte på
{"--mdns"} Aktiver mDNS-oppdagelse
{"--cors"} Ytterligere nettleseropprinnelse som tillater CORS

export

Eksporter øktdata som JSON.

closedcode export [sessionID]

Hvis du ikke gir en økt ID, blir du bedt om å velge fra tilgjengelige økter.

import

Importer øktdata fra en JSON-fil eller en vanilla-closedcode share-URL.

closedcode import <file>

Du kan importere fra en lokal fil eller en vanilla-closedcode share-URL.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

closedcode web

Dette starter en HTTP-server og åpner en nettleser for å få tilgang til vanilla-closedcode via et nettgrensesnitt. Sett CLOSEDCODE_SERVER_PASSWORD for å aktivere HTTP grunnleggende autentisering (brukernavn er standard til closedcode).
