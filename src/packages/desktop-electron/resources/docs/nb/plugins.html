Programtillegg

Skriv dine egne programtillegg for å utvide vanilla-closedcode.

Plugins lar deg utvide vanilla-closedcode ved å koble til ulike hendelser og tilpasse atferd. Du kan lage plugins for å legge til nye funksjoner, integrere med eksterne tjenester eller endre standardoppførselen til vanilla-closedcode.

For eksempler, sjekk ut plugins opprettet av fellesskapet.

Bruk av en plugin

Det er to måter å laste inn plugins.

Fra lokale filer

Plasser JavaScript- eller TypeScript-filer i plugin-katalogen.

  • .closedcode/plugins/ - Programtillegg på prosjektnivå
  • ~/.config/closedcode/plugins/ - Globale plugins

Filer i disse katalogene lastes automatisk ved oppstart.

Fra npm

Spesifiser npm-pakker i konfigurasjonsfilen.

{
  "plugin": ["opencode-helicone-session", "opencode-wakatime", "@my-org/custom-plugin"]
}

Både vanlige og scoped npm-pakker støttes.

Bla gjennom tilgjengelige plugins i økosystemet.

Hvordan plugins installeres

npm-plugins installeres automatisk ved hjelp av Node.js ved oppstart. Pakker og deres avhengigheter er bufret i ~/.cache/closedcode/node_modules/.

Lokale plugins lastes direkte fra plugin-katalogen. For å bruke eksterne pakker, må du opprette en package.json i konfigurasjonskatalogen din (se Dependencies), eller publisere plugin-en til npm og legg den til i konfigurasjonen din.

Lasterekkefølge

Plugins lastes inn fra alle kilder og alle kroker kjøres i rekkefølge. Lastrekkefølgen er:

  1. Global konfigurasjon (~/.config/closedcode/closedcode.json)
  2. Prosjektkonfigurasjon (closedcode.json)
  3. Global plugin-katalog (~/.config/closedcode/plugins/)
  4. Prosjektpluginkatalog (.closedcode/plugins/)

Dupliserte npm-pakker med samme navn og versjon lastes inn én gang. Imidlertid lastes en lokal plugin og en npm plugin med lignende navn begge separat.

Opprette en plugin

En plugin er en JavaScript/TypeScript-modul som eksporterer en eller flere plugin-moduler funksjoner. Hver funksjon mottar et kontekstobjekt og returnerer et krokobjekt.

Avhengigheter

Lokale plugins og tilpassede verktøy kan bruke eksterne npm-pakker. Legg til en package.json til konfigurasjonskatalogen med avhengighetene du trenger.

{
  "dependencies": {
    "shescape": "^2.1.0"
  }
}

vanilla-closedcode kjører npm install ved oppstart for å installere disse. Programtilleggene og verktøyene dine kan deretter importere dem.




  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        output.args.command = escape(output.args.command)
      }
    },
  }
}

Grunnleggende struktur


  console.log("Plugin initialized!")

  return {
    // Hook implementations go here
  }
}

Plugin-funksjonen mottar:

  • project: Gjeldende prosjektinformasjon.
  • directory: Gjeldende arbeidskatalog.
  • worktree: Git-arbeidstrebanen.
  • client: En vanilla-closedcode SDK klient for samhandling med AI.

TypeScript-støtte

For TypeScript-plugins kan du importere typer fra plugin-pakken:




  return {
    // Type-safe hook implementations
  }
}

Eksempler

Her er noen eksempler på plugins du kan bruke for å utvide vanilla-closedcode.

Send varsler

Send varsler når visse hendelser inntreffer:


  return {
    event: async ({ event }) => {
      // Send notification on session completion
      if (event.type === "session.idle") {
        await $`osascript -e 'display notification "Session completed!" with title "closedcode"'`
      }
    },
  }
}

Vi bruker osascript for å kjøre AppleScript på macOS. Her bruker vi den til å sende varsler.

Hvis du bruker vanilla-closedcode-skrivebordsappen, kan den sende systemvarsler automatisk når et svar er klart eller når en økt feiler.
---

.env-beskyttelse

Hindre vanilla-closedcode fra å lese .env filer:


  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "read" && output.args.filePath.includes(".env")) {
        throw new Error("Do not read .env files")
      }
    },
  }
}

Injiser miljøvariabler

Injiser miljøvariabler i all shell-utførelse (AI-verktøy og brukerterminaler):


  return {
    "shell.env": async (input, output) => {
      output.env.MY_API_KEY = "secret"
      output.env.PROJECT_ROOT = input.cwd
    },
  }
}

Egendefinerte verktøy

Plugins kan også legge til egendefinerte verktøy til vanilla-closedcode:




  return {
    tool: {
      mytool: tool({
        description: "This is a custom tool",
        args: {
          foo: tool.schema.string(),
        },
        async execute(args, context) {
          const { directory, worktree } = context
          return `Hello ${args.foo} from ${directory} (worktree: ${worktree})`
        },
      }),
    },
  }
}

tool-hjelperen lager et tilpasset verktøy som vanilla-closedcode kan kalle. Den tar en Zod-skjemafunksjon og returnerer en verktøydefinisjon med:

  • description: Hva verktøyet gjør
  • args: Zod-skjema for verktøyets argumenter
    • execute: Funksjon som kjører når verktøyet kalles

Dine egendefinerte verktøy vil være tilgjengelige for vanilla-closedcode sammen med innebygde verktøy.

Hvis et plugin-verktøy bruker samme navn som et innebygd verktøy, vil plugin-verktøyet ha forrang.
---

Logging

Bruk client.app.log() i stedet for console.log for strukturert logging:


  await client.app.log({
    body: {
      service: "my-plugin",
      level: "info",
      message: "Plugin initialized",
      extra: { foo: "bar" },
    },
  })
}

Nivåer: debug, info, warn, error. Se SDK dokumentasjon for detaljer.

Komprimerings-hooks

Tilpass konteksten inkludert når en økt komprimeres:




  return {
    "experimental.session.compacting": async (input, output) => {
      // Inject additional context into the compaction prompt
      output.context.push(`
## Custom Context

Include any state that should persist across compaction:
- Current task status
- Important decisions made
- Files being actively worked on
`)
    },
  }
}

experimental.session.compacting-kroken trigges før LLM genererer et fortsettelsessammendrag. Bruk den til å injisere domenespesifikk kontekst som standard komprimeringsprompt ville gå glipp av.

Du kan også erstatte komprimeringsprompten helt ved å stille inn output.prompt:




  return {
    "experimental.session.compacting": async (input, output) => {
      // Replace the entire compaction prompt
      output.prompt = `
You are generating a continuation prompt for a multi-agent swarm session.

Summarize:
1. The current task and its status
2. Which files are being modified and by whom
3. Any blockers or dependencies between agents
4. The next steps to complete the work

Format as a structured prompt that a new agent can use to resume work.
`
    },
  }
}

Når output.prompt er angitt, erstatter den standard komprimeringsprompt fullstendig. output.context-matrisen ignoreres i dette tilfellet.
