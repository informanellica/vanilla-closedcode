Windows (WSL)

WSL を使って Windows で vanilla-closedcode を使う。

vanilla-closedcode は Windows で直接実行できますが、より快適に使うには Windows Subsystem for Linux (WSL) の利用をおすすめします。WSL は vanilla-closedcode の機能とスムーズに連携する Linux 環境を提供します。

WSL を使う理由

WSL を使うと、ファイルシステム性能、ターミナルサポート、vanilla-closedcode が依存する開発ツールとの互換性が向上します。
---

セットアップ

  1. WSL をインストールする

    まだの場合は、Microsoft 公式ガイドを使って WSL をインストール します。

  2. WSL で vanilla-closedcode をインストールする

    WSL の準備ができたら WSL のターミナルを開き、インストール方法 のいずれかで vanilla-closedcode をインストールします。

    リリースページ https://github.com/informanellica/vanilla-closedcode/releases から最新版をダウンロードしてインストールしてください。

  3. WSL から vanilla-closedcode を使う

    プロジェクトディレクトリに移動し（Windows ファイルは /mnt/c/ や /mnt/d/ などからアクセス）、vanilla-closedcode を実行します。

    cd /mnt/c/Users/YourName/project
closedcode

デスクトップアプリ + WSL サーバー

vanilla-closedcode デスクトップアプリを使いつつ、サーバーは WSL で動かしたい場合は次の手順です。

  1. WSL でサーバーを起動する

    外部接続を許可するため、--hostname 0.0.0.0 を付けて起動します。

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. デスクトップアプリを接続する

    http://localhost:4096 に接続します。

環境によって `localhost` が使えない場合は、WSL 側で `hostname -I` を実行して IP アドレスを確認し、`http://:4096` に接続してください。
`--hostname 0.0.0.0` を使う場合は、`CLOSEDCODE_SERVER_PASSWORD` を設定してサーバーを保護してください。 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
## Web クライアント + WSL

Windows で Web 利用を快適にするには:

  1. PowerShell ではなく WSL ターミナルで closedcode web を実行する

    closedcode web --hostname 0.0.0.0

  2. Windows のブラウザからアクセスする

    http://localhost:<port> にアクセスします（URL は vanilla-closedcode が表示します）。

WSL から closedcode web を実行すると、適切なファイルシステムアクセスとターミナル統合を維持したまま、Windows ブラウザから利用できます。

Windows ファイルへのアクセス

WSL からは /mnt/ ディレクトリ経由で Windows ファイルにアクセスできます。

  • C: drive → /mnt/c/
  • D: drive → /mnt/d/
  • そのほかのドライブも同様です

例:

cd /mnt/c/Users/YourName/Documents/project
closedcode
よりスムーズに使うには、リポジトリを WSL のファイルシステム（例: `~/code/`）にクローンまたはコピーして、そこで vanilla-closedcode を実行することをおすすめします。
---

ヒント

  • Windows ドライブ上のプロジェクトでも、vanilla-closedcode は WSL で実行するとファイルアクセスがスムーズです
  • vanilla-closedcode と一緒に VS Code の WSL 拡張機能 を使うと統合的な開発フローを構築できます
  • vanilla-closedcode の設定とセッションは WSL 環境内の ~/.local/share/closedcode/ に保存されます
