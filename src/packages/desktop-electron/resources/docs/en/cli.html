CLI

vanilla-closedcode CLI options and commands.

The vanilla-closedcode CLI by default starts the TUI when run without any arguments.

closedcode

But it also accepts commands as documented on this page. This allows you to interact with vanilla-closedcode programmatically.

closedcode run "Explain how closures work in JavaScript"

tui

Start the vanilla-closedcode terminal user interface.

closedcode [project]

Flags

Flag Short Description
{"--continue"} -c Continue the last session
{"--session"} -s Session ID to continue
{"--fork"} Fork the session when continuing (use with --continue or --session)
{"--prompt"} Prompt to use
{"--model"} -m Model to use in the form of provider/model
{"--agent"} Agent to use
{"--port"} Port to listen on
{"--hostname"} Hostname to listen on
{"--mdns"} Enable mDNS discovery
{"--mdns-domain"} Custom mDNS domain name
{"--cors"} Additional browser origin(s) to allow CORS

Commands

The vanilla-closedcode CLI also has the following commands.

agent

Manage agents for vanilla-closedcode.

closedcode agent [command]

attach

Attach a terminal to an already running vanilla-closedcode backend server started via serve or web commands.

closedcode attach [url]

This allows using the TUI with a remote vanilla-closedcode backend. For example:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Flags

Flag Short Description
{"--dir"} Working directory to start TUI in
{"--continue"} -c Continue the last session
{"--session"} -s Session ID to continue
{"--fork"} Fork the session when continuing (use with --continue or --session)
{"--password"} -p Basic auth password (defaults to CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Basic auth username (defaults to CLOSEDCODE_SERVER_USERNAME or closedcode)

create

Create a new agent with custom configuration.

closedcode agent create

This command will guide you through creating a new agent with a custom system prompt and permission configuration. Anything you don't allow is denied in the generated agent's frontmatter.

Flags

Flag Short Description
{"--path"} Directory to write the agent file to (defaults to global or .closedcode/agent based on the prompt)
{"--description"} What the agent should do
{"--mode"} Agent mode: all, primary, or subagent
{"--permissions"} Comma-separated list of permissions to allow (default: all). Available: bash, read, edit, glob, grep, webfetch, task, todowrite, websearch, lsp, skill. Anything omitted is denied. Alias: --tools
{"--model"} -m Model to use, in provider/model format

Passing all of --path, --description, --mode, and --permissions runs the command non-interactively.

list

List all available agents.

closedcode agent list

auth

Command to manage credentials and login for providers.

closedcode auth [command]

login

vanilla-closedcode is powered by the provider list at Models.dev, so you can use closedcode auth login to configure API keys for any provider you'd like to use. This is stored in ~/.local/share/closedcode/auth.json.

closedcode auth login

When vanilla-closedcode starts up it loads the providers from the credentials file. And if there are any keys defined in your environments or a .env file in your project.

Flags
Flag Short Description
{"--provider"} -p Provider ID or name to log in to
{"--method"} -m Login method label to use, skipping method selection

list

Lists all the authenticated providers as stored in the credentials file.

closedcode auth list

Or the short version.

closedcode auth ls

logout

Logs you out of a provider by clearing it from the credentials file.

closedcode auth logout

github

Manage the GitHub agent for repository automation.

closedcode github [command]

install

Install the GitHub agent in your repository.

closedcode github install

This sets up the necessary GitHub Actions workflow and guides you through the configuration process. Learn more.

run

Run the GitHub agent. This is typically used in GitHub Actions.

closedcode github run
Flags
Flag Description
{"--event"} GitHub mock event to run the agent for
{"--token"} GitHub personal access token

mcp

Manage Model Context Protocol servers.

closedcode mcp [command]

add

Add an MCP server to your configuration.

closedcode mcp add

This command will guide you through adding either a local or remote MCP server.

list

List all configured MCP servers and their connection status.

closedcode mcp list

Or use the short version.

closedcode mcp ls

auth

Authenticate with an OAuth-enabled MCP server.

closedcode mcp auth [name]

If you don't provide a server name, you'll be prompted to select from available OAuth-capable servers.

You can also list OAuth-capable servers and their authentication status.

closedcode mcp auth list

Or use the short version.

closedcode mcp auth ls

logout

Remove OAuth credentials for an MCP server.

closedcode mcp logout [name]

debug

Debug OAuth connection issues for an MCP server.

closedcode mcp debug <name>

models

List all available models from configured providers.

closedcode models [provider]

This command displays all models available across your configured providers in the format provider/model.

This is useful for figuring out the exact model name to use in your config.

You can optionally pass a provider ID to filter models by that provider.

closedcode models anthropic

Flags

Flag Description
{"--refresh"} Refresh the models cache from models.dev
{"--verbose"} Use more verbose model output (includes metadata like costs)

Use the --refresh flag to update the cached model list. This is useful when new models have been added to a provider and you want to see them in vanilla-closedcode.

closedcode models --refresh

run

Run closedcode in non-interactive mode by passing a prompt directly.

closedcode run [message..]

This is useful for scripting, automation, or when you want a quick answer without launching the full TUI. For example.

closedcode run Explain the use of context in Go

You can also attach to a running closedcode serve instance to avoid MCP server cold boot times on every run:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Flags

Flag Short Description
{"--command"} The command to run, use message for args
{"--continue"} -c Continue the last session
{"--session"} -s Session ID to continue
{"--fork"} Fork the session when continuing (use with --continue or --session)
{"--share"} Share the session
{"--model"} -m Model to use in the form of provider/model
{"--agent"} Agent to use
{"--file"} -f File(s) to attach to message
{"--format"} Format: default (formatted) or json (raw JSON events)
{"--title"} Title for the session (uses truncated prompt if no value provided)
{"--attach"} Attach to a running closedcode server (e.g., http://localhost:4096)
{"--password"} -p Basic auth password (defaults to CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Basic auth username (defaults to CLOSEDCODE_SERVER_USERNAME or closedcode)
{"--dir"} Directory to run in, or path on the remote server when attaching
{"--port"} Port for the local server (defaults to random port)
{"--variant"} Model variant (provider-specific reasoning effort)
{"--thinking"} Show thinking blocks
{"--dangerously-skip-permissions"} Auto-approve permissions that are not explicitly denied (dangerous!)

serve

Start a headless vanilla-closedcode server for API access. Check out the server docs for the full HTTP interface.

closedcode serve

This starts an HTTP server that provides API access to closedcode functionality without the TUI interface. Set CLOSEDCODE_SERVER_PASSWORD to enable HTTP basic auth (username defaults to closedcode).

Flags

Flag Description
{"--port"} Port to listen on
{"--hostname"} Hostname to listen on
{"--mdns"} Enable mDNS discovery
{"--mdns-domain"} Custom mDNS domain name
{"--cors"} Additional browser origin(s) to allow CORS

session

Manage vanilla-closedcode sessions.

closedcode session [command]

list

List all vanilla-closedcode sessions.

closedcode session list
Flags
Flag Short Description
{"--max-count"} -n Limit to N most recent sessions
{"--format"} Output format: table or json (table)

delete

Delete an vanilla-closedcode session.

closedcode session delete <sessionID>

stats

Show token usage and cost statistics for your vanilla-closedcode sessions.

closedcode stats

Flags

Flag Description
{"--days"} Show stats for the last N days (all time)
{"--tools"} Number of tools to show (all)
{"--models"} Show model usage breakdown (hidden by default). Pass a number to show top N
{"--project"} Filter by project (all projects, empty string: current project)

export

Export session data as JSON.

closedcode export [sessionID]

If you don't provide a session ID, you'll be prompted to select from available sessions.

Flags

Flag Description
{"--sanitize"} Redact sensitive transcript/file data

import

Import session data from a JSON file or vanilla-closedcode share URL.

closedcode import <file>

You can import from a local file or an vanilla-closedcode share URL.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

Start a headless vanilla-closedcode server with a web interface.

closedcode web

This starts an HTTP server and opens a web browser to access vanilla-closedcode through a web interface. Set CLOSEDCODE_SERVER_PASSWORD to enable HTTP basic auth (username defaults to closedcode).

Flags

Flag Description
{"--port"} Port to listen on
{"--hostname"} Hostname to listen on
{"--mdns"} Enable mDNS discovery
{"--mdns-domain"} Custom mDNS domain name
{"--cors"} Additional browser origin(s) to allow CORS

acp

Start an ACP (Agent Client Protocol) server.

closedcode acp

This command starts an ACP server that communicates via stdin/stdout using nd-JSON.

Flags

Flag Description
{"--cwd"} Working directory
{"--port"} Port to listen on
{"--hostname"} Hostname to listen on
{"--mdns"} Enable mDNS discovery
{"--mdns-domain"} Custom mDNS domain name
{"--cors"} Additional browser origin(s) to allow CORS

plugin

Install a plugin and update your config.

closedcode plugin <module>

Or use the alias.

closedcode plug <module>

Flags

Flag Short Description
{"--global"} -g Install in global config
{"--force"} -f Replace existing plugin version

pr

Fetch and checkout a GitHub PR branch, then run vanilla-closedcode.

closedcode pr <number>

db

Database tools.

closedcode db [query]

Flags

Flag Description
{"--format"} Output format: json or tsv

path

Print the database path.

closedcode db path

debug

Debugging and troubleshooting tools.

closedcode debug [command]

uninstall

Uninstall vanilla-closedcode and remove all related files.

closedcode uninstall

Flags

Flag Short Description
{"--keep-config"} -c Keep configuration files
{"--keep-data"} -d Keep session data and snapshots
{"--dry-run"} Show what would be removed without removing
{"--force"} -f Skip confirmation prompts

upgrade

Updates closedcode to the latest version or a specific version.

closedcode upgrade [target]

To upgrade to the latest version.

closedcode upgrade

To upgrade to a specific version.

closedcode upgrade v0.1.48

Flags

Flag Short Description
{"--method"} -m The installation method that was used; curl, npm, pnpm, brew

Environment variables

vanilla-closedcode can be configured using environment variables.

Variable Type Description
CLOSEDCODE_AUTO_SHARE boolean Automatically share sessions
CLOSEDCODE_GIT_BASH_PATH string Path to Git Bash executable on Windows
CLOSEDCODE_CONFIG string Path to config file
CLOSEDCODE_TUI_CONFIG string Path to TUI config file
CLOSEDCODE_CONFIG_DIR string Path to config directory
CLOSEDCODE_CONFIG_CONTENT string Inline json config content
CLOSEDCODE_DISABLE_AUTOUPDATE boolean Disable automatic update checks
CLOSEDCODE_DISABLE_PRUNE boolean Disable pruning of old data
CLOSEDCODE_DISABLE_TERMINAL_TITLE boolean Disable automatic terminal title updates
CLOSEDCODE_PERMISSION string Inlined json permissions config
CLOSEDCODE_DISABLE_DEFAULT_PLUGINS boolean Disable default plugins
CLOSEDCODE_DISABLE_LSP_DOWNLOAD boolean Disable automatic LSP server downloads
CLOSEDCODE_ENABLE_EXPERIMENTAL_MODELS boolean Enable experimental models
CLOSEDCODE_DISABLE_AUTOCOMPACT boolean Disable automatic context compaction
CLOSEDCODE_DISABLE_CLAUDE_CODE boolean Disable reading from .claude (prompt + skills)
CLOSEDCODE_DISABLE_CLAUDE_CODE_PROMPT boolean Disable reading ~/.claude/CLAUDE.md
CLOSEDCODE_DISABLE_CLAUDE_CODE_SKILLS boolean Disable loading .claude/skills
CLOSEDCODE_DISABLE_MODELS_FETCH boolean Disable fetching models from remote sources
CLOSEDCODE_DISABLE_MOUSE boolean Disable mouse capture in the TUI
CLOSEDCODE_FAKE_VCS string Fake VCS provider for testing purposes
CLOSEDCODE_CLIENT string Client identifier (defaults to cli)
CLOSEDCODE_ENABLE_EXA boolean Enable Exa web search tools
CLOSEDCODE_SERVER_PASSWORD string Enable basic auth for serve/web
CLOSEDCODE_SERVER_USERNAME string Override basic auth username (default closedcode)
CLOSEDCODE_MODELS_URL string Custom URL for fetching models configuration
