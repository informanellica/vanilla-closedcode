CLI

Параметры и команда closedcode CLI.

CLI closedcode по умолчанию запускает TUI при запуске без каких-либо аргументов.

closedcode

Но он также принимает команды, описанные на этой странице. Это позволяет вам программно взаимодействовать с closedcode.

closedcode run "Explain how closures work in JavaScript"

tui

Запустите TUI closedcode.

closedcode [project]

Флаги

Флаг Короткий Описание
{"--continue"} -c Продолжить последний сеанс
{"--session"} -s Идентификатор сеанса для продолжения
{"--fork"} Разветвить сеанс при продолжении (используйте с --continue или --session)
{"--prompt"} Промпт для использования
{"--model"} -m Модель для использования в виде поставщика/модели.
{"--agent"} Агент для использования
{"--port"} Порт для прослушивания
{"--hostname"} Имя хоста для прослушивания

Команды

CLI closedcode также имеет следующие команды.

agent

Управляйте агентами для closedcode.

closedcode agent [command]

attach

Подключите терминал к уже работающему внутреннему серверу closedcode, запущенному с помощью команд serve или web.

closedcode attach [url]

Это позволяет использовать TUI с удаленным сервером closedcode. Например:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Флаги

Флаг Короткий Описание
{"--dir"} Рабочий каталог для запуска TUI
{"--continue"} -c Продолжить последний сеанс
{"--session"} -s Идентификатор сеанса для продолжения
{"--fork"} Создать ответвление сеанса при продолжении (используйте с --continue или --session)
{"--password"} -p Пароль базовой аутентификации (по умолчанию CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Имя пользователя базовой аутентификации (по умолчанию CLOSEDCODE_SERVER_USERNAME или closedcode)

create

Создайте нового агента с пользовательской конфигурацией.

closedcode agent create

Эта команда поможет вам создать новый агент с настраиваемой системной подсказкой и настройкой инструмента.

list

Перечислите всех доступных агентов.

closedcode agent list

auth

Команда для управления учетными данными и входом в систему для провайдеров.

closedcode auth [command]

login

closedcode использует список провайдеров с Models.dev, поэтому вы можете использовать closedcode auth login для настройки ключей API для любого поставщика, которого вы хотите использовать. Это хранится в ~/.local/share/closedcode/auth.json.

closedcode auth login

Когда closedcode запускается, он загружает поставщиков из файла учетных данных. И если в ваших средах определены какие-либо ключи или файл .env в вашем проекте.

list

Перечисляет всех проверенных поставщиков, которые хранятся в файле учетных данных.

closedcode auth list

Или короткая версия.

closedcode auth ls

logout

Выключает вас из провайдера, удаляя его из файла учетных данных.

closedcode auth logout

github

Управляйте агентом GitHub для автоматизации репозитория.

closedcode github [command]

install

Установите агент GitHub в свой репозиторий.

closedcode github install

Это настроит необходимый рабочий процесс GitHub Actions и проведет вас через процесс настройки. Подробнее.

run

Запустите агент GitHub. Обычно это используется в действиях GitHub.

closedcode github run
Флаги
Флаг Описание
{"--event"} Имитирующее событие GitHub для запуска агента
{"--token"} Токен личного доступа GitHub

mcp

Управляйте серверами протокола контекста модели.

closedcode mcp [command]

add

Добавьте сервер MCP в свою конфигурацию.

closedcode mcp add

Эта команда поможет вам добавить локальный или удаленный сервер MCP.

list

Перечислите все настроенные серверы MCP и состояние их подключения.

closedcode mcp list

Или используйте короткую версию.

closedcode mcp ls

auth

Аутентификация с помощью сервера MCP с поддержкой OAuth.

closedcode mcp auth [name]

Если вы не укажете имя сервера, вам будет предложено выбрать один из доступных серверов с поддержкой OAuth.

Вы также можете перечислить серверы с поддержкой OAuth и их статус аутентификации.

closedcode mcp auth list

Или используйте короткую версию.

closedcode mcp auth ls

logout

Удалите учетные данные OAuth для сервера MCP.

closedcode mcp logout [name]

debug

Отладка проблем с подключением OAuth для сервера MCP.

closedcode mcp debug <name>

models

Перечислите все доступные модели от настроенных поставщиков.

closedcode models [provider]

Эта команда отображает все модели, доступные у настроенных вами поставщиков, в формате provider/model.

Это полезно для определения точного названия модели, которое будет использоваться в вашем config.

При желании вы можете передать идентификатор поставщика, чтобы фильтровать модели по этому поставщику.

closedcode models anthropic

Флаги

Флаг Описание
{"--refresh"} Обновите кеш моделей на сайте models.dev.
{"--verbose"} Используйте более подробный вывод модели (включая метаданные, такие как затраты).

Используйте флаг --refresh для обновления списка кэшированных моделей. Это полезно, когда к поставщику добавлены новые модели и вы хотите увидеть их в closedcode.

closedcode models --refresh

run

Запустите closedcode в неинтерактивном режиме, передав приглашение напрямую.

closedcode run [message..]

Это полезно для создания сценариев, автоматизации или когда вам нужен быстрый ответ без запуска полного TUI. Например.

closedcode run Explain the use of context in Go

Вы также можете подключиться к работающему экземпляру closedcode serve, чтобы избежать холодной загрузки сервера MCP при каждом запуске:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Флаги

Флаг Короткий Описание
{"--command"} Команда для запуска, используйте сообщение для аргументов
{"--continue"} -c Продолжить последний сеанс
{"--session"} -s Идентификатор сеанса для продолжения
{"--fork"} Разветвить сеанс при продолжении (используйте с --continue или --session)
{"--share"} Поделиться сеансом

{"--format"} Формат: по умолчанию (отформатированный) или json (необработанные события JSON).
{"--title"} Название сеанса (использует усеченное приглашение, если значение не указано)
{"--attach"} Подключитесь к работающему серверу closedcode (например, http://localhost:4096)
{"--password"} -p Пароль базовой аутентификации (по умолчанию CLOSEDCODE_SERVER_PASSWORD)

{"--thinking"} Показать блоки размышлений
{"--port"} Порт локального сервера (по умолчанию случайный порт)

serve

Запустите автономный сервер closedcode для доступа к API. Полный HTTP-интерфейс можно найти в server docs.

closedcode serve

При этом запускается HTTP-сервер, который обеспечивает доступ API к функциям closedcode без интерфейса TUI. Установите CLOSEDCODE_SERVER_PASSWORD, чтобы включить базовую аутентификацию HTTP (имя пользователя по умолчанию — closedcode).

Флаги

Флаг Описание
{"--port"} Порт для прослушивания
{"--hostname"} Имя хоста для прослушивания
{"--mdns"} Включить обнаружение mDNS
{"--cors"} Дополнительные источники браузера, позволяющие разрешить CORS

session

Управляйте сессиями closedcode.

closedcode session [command]

list

Перечислите все сеансы closedcode.

closedcode session list
Флаги
Флаг Короткий Описание
{"--max-count"} -n Ограничить N последних сеансов.
{"--format"} Формат вывода: таблица или json (таблица)

stats

Покажите статистику использования токенов и затрат для ваших сеансов closedcode.

closedcode stats

Флаги

Флаг Описание
{"--days"} Показать статистику за последние N дней (все время)
{"--tools"} Количество инструментов для отображения (все)
{"--models"} Показать разбивку по использованию модели (по умолчанию скрыто). Передайте номер, чтобы показать верхнюю N
{"--project"} Фильтровать по проекту (все проекты, пустая строка: текущий проект)

export

Экспортируйте данные сеанса в формате JSON.

closedcode export [sessionID]

Если вы не укажете идентификатор сеанса, вам будет предложено выбрать один из доступных сеансов.

import

Импортируйте данные сеанса из файла JSON или URL-адреса общего ресурса closedcode.

closedcode import <file>

Вы можете импортировать из локального файла или URL-адреса общего ресурса closedcode.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

Запустите автономный сервер closedcode с веб-интерфейсом.

closedcode web

При этом запускается HTTP-сервер и открывается веб-браузер для доступа к closedcode через веб-интерфейс. Установите CLOSEDCODE_SERVER_PASSWORD, чтобы включить базовую аутентификацию HTTP (имя пользователя по умолчанию — closedcode).

Флаги

Флаг Описание
{"--port"} Порт для прослушивания
{"--hostname"} Имя хоста для прослушивания
{"--mdns"} Включить обнаружение mDNS
{"--cors"} Дополнительные источники браузера, позволяющие разрешить CORS

acp

Запустите сервер ACP (агент-клиентский протокол).

closedcode acp

Эта команда запускает сервер ACP, который обменивается данными через stdin/stdout с использованием nd-JSON.

Флаги

Флаг Описание

{"--port"} Порт для прослушивания
{"--hostname"} Имя хоста для прослушивания
