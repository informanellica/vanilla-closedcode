Windows (WSL)

Запускайте closedcode в Windows через WSL.

closedcode можно запускать напрямую в Windows, но для лучшего опыта мы рекомендуем Windows Subsystem for Linux (WSL). WSL дает Linux-среду, которая отлично работает с возможностями closedcode.

Почему WSL?

WSL дает более высокую производительность файловой системы, полноценную поддержку терминала и совместимость с инструментами разработки, на которые опирается closedcode.
---

Настройка

  1. Установите WSL

    Если вы еще не сделали этого, установите WSL по официальному руководству Microsoft.

  2. Установите closedcode в WSL

    После настройки WSL откройте терминал WSL и установите closedcode одним из способов установки.

    Скачайте последнюю версию со страницы релизов: https://github.com/informanellica/vanilla-closedcode/releases

  3. Запускайте closedcode из WSL

    Перейдите в каталог проекта (к файлам Windows можно обращаться через /mnt/c/, /mnt/d/ и т.д.) и запустите closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

Десктопное приложение + сервер в WSL

Если вы предпочитаете closedcode Desktop, но хотите запускать сервер в WSL:

  1. Запустите сервер в WSL с параметром --hostname 0.0.0.0, чтобы разрешить внешние подключения:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Подключите десктопное приложение к http://localhost:4096

Если в вашей конфигурации `localhost` не работает, используйте IP-адрес WSL (выполните в WSL: `hostname -I`) и подключайтесь по `http://:4096`.
При использовании `--hostname 0.0.0.0` задайте `CLOSEDCODE_SERVER_PASSWORD`, чтобы защитить сервер. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Веб-клиент + WSL

Для лучшего веб-опыта в Windows:

  1. Запускайте closedcode web в терминале WSL, а не в PowerShell:

    closedcode web --hostname 0.0.0.0

  2. Открывайте в браузере Windows адрес http://localhost:<port> (closedcode выведет URL)

Запуск closedcode web из WSL обеспечивает корректный доступ к файловой системе и интеграцию с терминалом, при этом интерфейс остается доступным из браузера Windows.

Доступ к файлам Windows

WSL может получать доступ ко всем вашим файлам Windows через каталог /mnt/:

  • C: drive → /mnt/c/
  • D: drive → /mnt/d/
  • И так далее

Пример:

cd /mnt/c/Users/YourName/Documents/project
closedcode
Для максимально плавной работы стоит клонировать или скопировать репозиторий в файловую систему WSL (например, в `~/code/`) и запускать closedcode оттуда.
---

Советы

  • Даже для проектов на дисках Windows запускайте closedcode в WSL, чтобы получить более плавный доступ к файлам
  • Используйте closedcode вместе с расширением WSL для VS Code для единого рабочего процесса
  • Конфигурация и сессии closedcode хранятся в среде WSL по пути ~/.local/share/closedcode/
