IDE

Die vanilla-closedcode-Erweiterung für VS Code, Cursor und andere IDEs

vanilla-closedcode lässt sich in VS Code, Cursor oder jede IDE integrieren, die ein Terminal unterstützt. Führen Sie einfach closedcode im Terminal aus, um zu beginnen.

Verwendung

  • Schnellstart: Verwenden Sie Cmd+Esc (Mac) oder Ctrl+Esc (Windows/Linux), um vanilla-closedcode in einer geteilten Terminalansicht zu öffnen, oder fokussieren Sie eine dieser Terminalsitzung, falls bereits eine ausgeführt wird.
  • Neue Sitzung: Verwenden Sie Cmd+Shift+Esc (Mac) oder Ctrl+Shift+Esc (Windows/Linux), um eine neue vanilla-closedcode-Terminalsitzung zu starten, auch wenn eine bereits geöffnet ist. Sie können auch auf die vanilla-closedcode-Schaltfläche im UI klicken.
  • Kontextbewusstsein: Teilen Sie Ihre aktuelle Auswahl oder Registerkarte automatisch mit vanilla-closedcode.
  • Verknüpfungen zu Dateiverweisen: Verwenden Sie Cmd+Option+K (Mac) oder Alt+Ctrl+K (Linux/Windows), um Dateiverweisen einzufügen. Beispiel: @File#L37-42.

Installation

So installieren Sie vanilla-closedcode in VS Code und beliebten Forks wie Cursor, Windsurf, VSCodium:

  1. Öffnen Sie VS Code
  2. Öffnen Sie das integrierte Terminal
  3. Führen Sie closedcode aus – die Erweiterung wird automatisch installiert

Wenn Sie andererseits Ihre eigene IDE verwenden möchten, wenn Sie /editor oder /export von TUI ausführen, müssen Sie export EDITOR="code --wait" festlegen. Learn more.

Manuelle Installation

Suchen Sie im Extension Marketplace nach vanilla-closedcode und klicken Sie auf Installieren.

Fehlerbehebung

Wenn die Erweiterung nicht automatisch installiert werden kann:

  • Stellen Sie sicher, dass Sie closedcode im integrierten Terminal ausführen.
  • Bestätigen Sie, dass CLI für Ihre IDE installiert ist:
    • Für VS Code: code Befehl
    • Für Cursor: Befehl cursor
    • Für Windsurf: Befehl windsurf
    • Für VSCodium: Befehl codium
    • Wenn nicht, führen Sie Cmd+Shift+P (Mac) oder Ctrl+Shift+P (Windows/Linux) aus und suchen Sie nach „Shell Command: Install 'code' command in PATH“ (oder dem Äquivalent für Ihre IDE).
  • Stellen Sie sicher, dass VS Code über die Berechtigung zum Installieren von Erweiterungen verfügt
