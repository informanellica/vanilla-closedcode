Windows (WSL)

Nutze vanilla-closedcode unter Windows am besten mit WSL.

vanilla-closedcode laeuft zwar direkt unter Windows, die beste Erfahrung bekommst du aber mit Windows Subsystem for Linux (WSL). WSL bietet eine Linux-Umgebung, die mit den vanilla-closedcode-Features nahtlos funktioniert.

Warum WSL?

WSL bietet bessere Dateisystem-Performance, vollstaendige Terminal-Unterstuetzung und hohe Kompatibilitaet mit den Entwicklungswerkzeugen, die vanilla-closedcode nutzt.
---

Einrichtung

  1. Install WSL

    Falls noch nicht geschehen, installiere WSL mit der offiziellen Microsoft-Anleitung.

  2. Install vanilla-closedcode in WSL

    Wenn WSL eingerichtet ist, oeffne dein WSL-Terminal und installiere vanilla-closedcode mit einer der Installationsmethoden.

    curl -fsSL https://github.com/informanellica/vanilla-closedcode/releases | bash

  3. vanilla-closedcode über WSL verwenden

    Wechsle in dein Projektverzeichnis (Windows-Dateien erreichst du ueber /mnt/c/, /mnt/d/ usw.) und starte vanilla-closedcode.

    cd /mnt/c/Users/YourName/project
closedcode

Desktop-App + WSL-Server

Wenn du die vanilla-closedcode-Desktop-App nutzen willst, den Server aber in WSL betreiben moechtest:

  1. Starte den Server in WSL mit --hostname 0.0.0.0, damit externe Verbindungen moeglich sind:

    closedcode serve --hostname 0.0.0.0 --port 4096

  2. Verbinde die Desktop-App mit http://localhost:4096

Wenn `localhost` in deiner Umgebung nicht funktioniert, nutze stattdessen die WSL-IP-Adresse (in WSL: `hostname -I`) und verbinde dich mit `http://:4096`.
Wenn du `--hostname 0.0.0.0` verwendest, setze `CLOSEDCODE_SERVER_PASSWORD`, um den Server zu schuetzen. 
CLOSEDCODE_SERVER_PASSWORD=your-password closedcode serve --hostname 0.0.0.0
---

Web-Client + WSL

Fuer die beste Web-Erfahrung unter Windows:

  1. Fuehre closedcode web im WSL-Terminal aus, nicht in PowerShell:

    closedcode web --hostname 0.0.0.0

  2. Rufe die URL im Windows-Browser auf unter http://localhost:<port> (vanilla-closedcode zeigt die URL an)

Wenn du closedcode web aus WSL startest, bleiben Dateizugriff und Terminal-Integration sauber. Gleichzeitig ist die Weboberflaeche weiterhin im Windows-Browser erreichbar.

Zugriff auf Windows-Dateien

WSL kann ueber das Verzeichnis /mnt/ auf alle Windows-Dateien zugreifen:

  • Laufwerk C: -> /mnt/c/
  • Laufwerk D: -> /mnt/d/
  • usw.

Beispiel:

cd /mnt/c/Users/YourName/Documents/project
closedcode
Fuer die beste Performance kannst du dein Repo direkt in das WSL-Dateisystem klonen/kopieren (z. B. nach `~/code/`) und vanilla-closedcode dort starten.
---

Tipps

  • Lass vanilla-closedcode in WSL laufen, auch wenn das Projekt auf einem Windows-Laufwerk liegt
  • Nutze die WSL-Extension von VS Code fuer einen integrierten Workflow
  • vanilla-closedcode-Konfiguration und Sitzungen liegen in WSL unter ~/.local/share/closedcode/
