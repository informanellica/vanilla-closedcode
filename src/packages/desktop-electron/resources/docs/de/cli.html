CLI

vanilla-closedcode CLI Optionen und Befehle.

Der vanilla-closedcode CLI startet standardmäßig das TUI, wenn er ohne Argumente ausgeführt wird.

closedcode

Es akzeptiert aber auch Befehle, wie auf dieser Seite dokumentiert. Dadurch können Sie programmgesteuert mit vanilla-closedcode interagieren.

closedcode run "Explain how closures work in JavaScript"

tui

Starten Sie die Terminalbenutzeroberfläche vanilla-closedcode.

closedcode [project]

Optionen

Flag Kurz Beschreibung
{"--continue"} -c Setzen Sie die letzte Sitzung fort
{"--session"} -s Session-ID zum Fortfahren
{"--fork"} Sitzung beim Fortsetzen verzweigen (mit --continue oder --session)
{"--prompt"} Prompt zur Verwendung
{"--model"} -m Zu verwendendes Modell in der Form provider/model
{"--agent"} Zu verwendender Agent
{"--port"} Port zum Abhören
{"--hostname"} Hostname zum Abhören

Befehle

Der vanilla-closedcode CLI verfügt außerdem über die folgenden Befehle.

agent

Agenten für vanilla-closedcode verwalten.

closedcode agent [command]

attach

Schließen Sie ein Terminal an einen bereits laufenden vanilla-closedcode-Backend-Server an, der über die Befehle serve oder web gestartet wurde.

closedcode attach [url]

Dies ermöglicht die Verwendung von TUI mit einem Remote-Backend vanilla-closedcode. Zum Beispiel:

# Start the backend server for web/mobile access
closedcode web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
closedcode attach http://10.20.30.40:4096

Optionen

Flag Kurz Beschreibung
{"--dir"} Arbeitsverzeichnis zum Starten von TUI in
{"--continue"} -c Letzte Sitzung fortsetzen
{"--session"} -s Session-ID zum Fortfahren
{"--fork"} Sitzung beim Fortsetzen abzweigen (mit --continue oder --session verwenden)
{"--password"} -p Basic-Auth-Passwort (standardmäßig CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Basic-Auth-Benutzername (standardmäßig CLOSEDCODE_SERVER_USERNAME oder closedcode)

create

Erstellen Sie einen neuen Agenten mit benutzerdefinierter Konfiguration.

closedcode agent create

Dieser Befehl führt Sie durch die Erstellung eines neuen Agenten mit einer benutzerdefinierten Systemeingabeaufforderung und Toolkonfiguration.

list

Listen Sie alle verfügbaren Agenten auf.

closedcode agent list

auth

Befehl zum Verwalten von Anmeldeinformationen und Anmelden für Anbieter.

closedcode auth [command]

login

vanilla-closedcode wird von der Anbieterliste bei Models.dev unterstützt, sodass Sie closedcode auth login verwenden können, um API-Schlüssel für jeden Anbieter zu konfigurieren, den Sie verwenden möchten. Dies wird in ~/.local/share/closedcode/auth.json gespeichert.

closedcode auth login

Wenn vanilla-closedcode startet, lädt es den Anbieter aus der Anmeldeinformationsdatei. Und ob in Ihren Umgebungen Schlüssel sind oder in Ihrem Projekt eine .env-Datei vorhanden ist.

list

Listet alle authentifizierten Anbieter auf, wie in der Anmeldeinformationsdatei gespeichert.

closedcode auth list

Oder die Kurzversion.

closedcode auth ls

logout

Melden Sie sich von einem Anbieter ab, indem Sie ihn aus der Anmeldeinformationsdatei löschen.

closedcode auth logout

github

Verwalten Sie den GitHub-Agenten für die Repository-Automatisierung.

closedcode github [command]

install

Installieren Sie den GitHub-Agenten in Ihrem Repository.

closedcode github install

Dadurch wird der erforderliche GitHub-Aktionsworkflow eingerichtet und Sie durch den Konfigurationsprozess geführt. Learn more.

run

Führen Sie den GitHub-Agenten aus. Dies wird normalerweise in GitHub-Aktionen verwendet.

closedcode github run
Optionen
Flag Beschreibung
{"--event"} GitHub Scheinereignis zum Ausführen des Agenten für
{"--token"} GitHub persönliches Zugriffstoken

mcp

Verwalten Sie Model Context Protocol-Server.

closedcode mcp [command]

add

Fügen Sie Ihrer Konfiguration einen MCP-Server hinzu.

closedcode mcp add

Dieser Befehl führt Sie durch das Hinzufügen eines lokalen oder Remote-MCP-Servers.

list

Listen Sie alle konfigurierten MCP-Server und deren Verbindungsstatus auf.

closedcode mcp list

Oder nutzen Sie die Kurzversion.

closedcode mcp ls

auth

Authentifizieren Sie sich mit einem OAuth-fähigen MCP-Server.

closedcode mcp auth [name]

Wenn Sie keinen Servernamen angeben, werden Sie aufgefordert, einen der verfügbaren OAuth-fähigen Server auszuwählen.

Sie können auch OAuth-fähige Server und deren Authentifizierungsstatus auflisten.

closedcode mcp auth list

Oder nutzen Sie die Kurzversion.

closedcode mcp auth ls

logout

Entfernen Sie OAuth-Anmeldeinformationen für einen MCP-Server.

closedcode mcp logout [name]

debug

Beheben Sie OAuth-Verbindungsprobleme für einen MCP-Server.

closedcode mcp debug <name>

models

Listen Sie alle verfügbaren Modelle der konfigurierten Anbieter auf.

closedcode models [provider]

Dieser Befehl zeigt alle bei Ihren konfigurierten Anbietern verfügbaren Modelle im Format provider/model an.

Dies ist nützlich, um die genauen Modellnamen herauszufinden, die in Ihrer Konfiguration verwendet werden sollen.

Sie können optional eine Anbieter-ID übergeben, um Modelle nach diesem Anbieter zu filtern.

closedcode models anthropic

Optionen

Flag Beschreibung
{"--refresh"} Aktualisieren Sie den Modellcache von models.dev
{"--verbose"} Verwenden Sie eine ausführlichere Modellausgabe (einschließlich Metadaten wie Kosten)

Verwenden Sie das Flag --refresh, um die zwischengespeicherte Modellliste zu aktualisieren. Dies ist nützlich, wenn einem Anbieter neue Modelle hinzugefügt wurden und Sie diese in vanilla-closedcode sehen möchten.

closedcode models --refresh

run

Führen Sie vanilla-closedcode im nicht interaktiven Modus aus, indem Sie direkt eine Eingabeaufforderung übergeben.

closedcode run [message..]

Dies ist nützlich für die Skripterstellung, die Automatisierung oder wenn Sie eine schnelle Antwort wünschen, ohne das vollständige TUI zu starten. Zum Beispiel:

closedcode run Explain the use of context in Go

Sie können auch eine Verbindung zu einer laufenden closedcode serve-Instanz herstellen, um bei jedem Lauf Kaltstartzeiten für den MCP-Server zu vermeiden:

# Start a headless server in one terminal
closedcode serve

# In another terminal, run commands that attach to it
closedcode run --attach http://localhost:4096 "Explain async/await in JavaScript"

Optionen

Flag Kurz Beschreibung
{"--command"} Der auszuführende Befehl, Argumente als Nachricht verwenden
{"--continue"} -c Setzen Sie die letzte Sitzung fort
{"--session"} -s Session-ID zum Fortfahren
{"--fork"} Sitzung beim Fortsetzen verzweigen (mit --continue oder --session verwenden)
{"--share"} Teilen Sie die Sitzung
{"--model"} -m Zu verwendendes Modell in der Form provider/model
{"--agent"} Zu verwendender Agent
{"--file"} -f Datei(en) zum Anhängen an die Nachricht
{"--format"} Format: default (formatiert) oder json (rohe JSON-Ereignisse)
{"--title"} Titel für die Sitzung (verwendet eine verkürzte Eingabeaufforderung, wenn kein Wert angegeben wird)
{"--attach"} An einen laufenden vanilla-closedcode-Server anschließen (z.B. http://localhost:4096)
{"--password"} -p Basic-Auth-Passwort (standardmäßig CLOSEDCODE_SERVER_PASSWORD)
{"--username"} -u Basic-Auth-Benutzername (standardmäßig CLOSEDCODE_SERVER_USERNAME oder closedcode)
{"--dir"} Verzeichnis für die Ausführung, oder Pfad auf dem Remote-Server beim Anhängen
{"--variant"} Modellvariante (anbieterspezifischer Reasoning-Aufwand)
{"--thinking"} Denkblöcke anzeigen
{"--port"} Port für den lokalen Server (standardmäßig zufälliger Port)

serve

Starten Sie einen Headless-vanilla-closedcode-Server für den API-Zugriff. Sehen Sie sich die Server-Dokumentation für die vollständige HTTP-Schnittstelle an.

closedcode serve

Dadurch wird ein HTTP-Server gestartet, der API-Zugriff auf vanilla-closedcode-Funktionalität ohne die TUI-Schnittstelle bietet. Legen Sie CLOSEDCODE_SERVER_PASSWORD fest, um die HTTP-Basisauthentifizierung zu aktivieren (Benutzername ist standardmäßig closedcode).

Optionen

Flag Beschreibung
{"--port"} Port zum Abhören
{"--hostname"} Hostname zum Abhören
{"--mdns"} mDNS-Discovery aktivieren
{"--cors"} Zusätzliche Browser-Ursprünge für CORS zulassen

session

vanilla-closedcode-Sitzungen verwalten.

closedcode session [command]

list

Listen Sie alle vanilla-closedcode-Sitzungen auf.

closedcode session list
Optionen
Flag Kurz Beschreibung
{"--max-count"} -n Beschränken auf die N neuesten Sitzungen
{"--format"} Ausgabeformat: table oder json (table)

stats

Zeigen Sie Token-Nutzungs- und Kostenstatistiken für Ihre vanilla-closedcode-Sitzungen an.

closedcode stats

Optionen

Flag Beschreibung
{"--days"} Statistiken für die letzten N Tage anzeigen (alle Zeiten)
{"--tools"} Anzahl der angebotenen Werkzeuge (alle)
{"--models"} Aufschlüsselung der Modellnutzung anzeigen (standardmäßig ausgeblendet). Übergeben Sie eine Zahl, um die obersten N anzuzeigen
{"--project"} Nach Projekt filtern (alle Projekte, leere Zeichenfolge: aktuelles Projekt)

export

Sitzungsdaten als JSON exportieren.

closedcode export [sessionID]

Wenn Sie keine Sitzungs-ID angeben, werden Sie aufgefordert, eine der verfügbaren Sitzungen auszuwählen.

import

Importieren Sie Sitzungsdaten aus einer JSON-Datei oder einer vanilla-closedcode-Freigabe-URL.

closedcode import <file>

Sie können aus einer lokalen Datei oder einer vanilla-closedcode-Freigabe-URL importieren.

closedcode import session.json
closedcode import https://share.example.com/s/abc123

web

Starten Sie einen Headless-vanilla-closedcode-Server mit einer Weboberfläche.

closedcode web

Dadurch wird ein HTTP-Server gestartet und ein Webbrowser geöffnet, um über eine Webschnittstelle auf vanilla-closedcode zuzugreifen. Legen Sie CLOSEDCODE_SERVER_PASSWORD fest, um die HTTP-Basisauthentifizierung zu aktivieren (Benutzername ist standardmäßig closedcode).

Optionen

Flag Beschreibung
{"--port"} Port zum Abhören
{"--hostname"} Hostname zum Abhören
{"--mdns"} mDNS-Discovery aktivieren
{"--cors"} Zusätzliche Browser-Ursprünge für CORS zulassen

acp

Starten Sie einen ACP (Agent Client Protocol) Server.

closedcode acp

Dieser Befehl startet einen ACP-Server, der über stdin/stdout unter Verwendung von nd-JSON kommuniziert.

Optionen

Flag Beschreibung
{"--cwd"} Arbeitsverzeichnis
{"--port"} Port zum Abhören
{"--hostname"} Hostname zum Abhören

uninstall

Deinstallieren Sie vanilla-closedcode und entfernen Sie alle zugehörigen Dateien.

closedcode uninstall

Optionen

Flag Kurz Beschreibung
{"--keep-config"} -c Konfigurationsdateien behalten
{"--keep-data"} -d Sitzungsdaten und Snapshots aufbewahren
{"--dry-run"} Zeigt, was entfernt werden würde, ohne zu entfernen
{"--force"} -f Bestätigungsaufforderungen überspringen

upgrade

Aktualisiert vanilla-closedcode auf die neueste Version oder eine bestimmte Version.

closedcode upgrade [target]

Um auf die neueste Version zu aktualisieren:

closedcode upgrade

Um auf eine bestimmte Version zu aktualisieren.

closedcode upgrade v0.1.48

Optionen

Flag Kurz Beschreibung
{"--method"} -m Die zu verwendende Installationsmethode; curl, npm, pnpm, brew

Umgebungsvariablen

vanilla-closedcode kann mithilfe von Umgebungsvariablen konfiguriert werden.

Variable Typ Beschreibung
CLOSEDCODE_AUTO_SHARE boolescher Wert Sitzungen automatisch teilen
CLOSEDCODE_GIT_BASH_PATH Zeichenfolge Pfad zur ausführbaren Git Bash-Datei unter Windows
CLOSEDCODE_CONFIG Zeichenfolge Pfad zur Konfigurationsdatei
CLOSEDCODE_CONFIG_DIR Zeichenfolge Pfad zum Konfigurationsverzeichnis
CLOSEDCODE_CONFIG_CONTENT Zeichenfolge Inline-JSON-Konfigurationsinhalt
CLOSEDCODE_DISABLE_AUTOUPDATE boolescher Wert Automatische Update-Prüfungen deaktivieren
CLOSEDCODE_DISABLE_PRUNE boolescher Wert Bereinigung alter Daten deaktivieren
CLOSEDCODE_DISABLE_TERMINAL_TITLE boolescher Wert Deaktivieren Sie die automatische Aktualisierung von Terminaltiteln
CLOSEDCODE_PERMISSION Zeichenfolge Inline-JSON-Berechtigungskonfiguration
CLOSEDCODE_DISABLE_DEFAULT_PLUGINS boolescher Wert Standard-Plugins deaktivieren
CLOSEDCODE_DISABLE_LSP_DOWNLOAD boolescher Wert Automatische LSP-Server-Downloads deaktivieren
CLOSEDCODE_ENABLE_EXPERIMENTAL_MODELS boolescher Wert Experimentelle Modelle aktivieren
CLOSEDCODE_DISABLE_AUTOCOMPACT boolescher Wert Automatische Kontextkomprimierung deaktivieren
CLOSEDCODE_DISABLE_CLAUDE_CODE boolescher Wert Deaktivieren Sie das Lesen von .claude (Prompt + Skills)
CLOSEDCODE_DISABLE_CLAUDE_CODE_PROMPT boolescher Wert Deaktivieren Sie das Lesen von ~/.claude/CLAUDE.md
CLOSEDCODE_DISABLE_CLAUDE_CODE_SKILLS boolescher Wert Deaktivieren Sie das Laden von .claude/skills
CLOSEDCODE_DISABLE_MODELS_FETCH boolescher Wert Deaktivieren Sie das Abrufen von Modellen aus Remote-Quellen
CLOSEDCODE_FAKE_VCS Zeichenfolge Gefälschter VCS-Anbieter zu Testzwecken
CLOSEDCODE_CLIENT Zeichenfolge Client-ID (standardmäßig cli)
CLOSEDCODE_ENABLE_EXA boolescher Wert Exa-Websuchtools aktivieren
CLOSEDCODE_SERVER_PASSWORD Zeichenfolge Aktivieren Sie die Basisauthentifizierung für serve/web
CLOSEDCODE_SERVER_USERNAME Zeichenfolge Benutzernamen für die Basisauthentifizierung überschreiben (Standard closedcode)
CLOSEDCODE_MODELS_URL Zeichenfolge Benutzerdefinierte URL zum Abrufen der Modellkonfiguration
